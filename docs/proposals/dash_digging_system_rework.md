# Dash / Digging System Rework Proposal

기준일: `2026-05-27`
상태: 설계 변경안

이 문서는 대시, 벽타기, 채굴 입력 체계를 재정리하기 위한 변경안을 정리한다.

핵심 방향:

```text
좌클릭 = 공격
우클릭 = 360도 배터리 대시
방향키 = 이동 / 벽 채굴 진입
벽타기 = 삭제
모래채굴 = 삭제, 추후 별도 시스템으로 재정의
```

---

## 1. 대시 / 벽타기 시스템 변경

### 1-1. 대시 입력 변경

대시는 기존 Shift가 아닌 마우스 우클릭으로 동작한다.

기존 대시 입력은 모두 삭제한다.

삭제 대상:

- 방향키 더블탭 대시
- `Z` 키 대시
- `Shift` 키 대시

### 1-2. 대시 방향

대시는 기존 캐릭터가 바라보는 방향이 아닌 마우스 위치를 향해 동작한다.

규칙:

- 플레이어 중심에서 마우스 위치 방향으로 대시한다.
- 360도 모든 방향으로 가능하다.
- 상향, 하향, 좌향, 우향, 대각선 방향 제한을 두지 않는다.

### 1-3. 대시 거리

대시 거리는 반드시 기존 구현 기준 거리인 `4U`만큼이다.

규칙:

- 대시 거리는 `4U`로 고정한다.
- 대시 방향과 무관하게 이동 거리는 동일하게 처리한다.
- 대시 거리 변경은 이번 작업 범위에 포함하지 않는다.

### 1-4. 대시 배터리 소모

대시는 배터리를 소모한다.

규칙:

- 대시 1회당 배터리를 `30` 소모한다.
- 배터리가 `30` 미만이면 대시할 수 없다.
- 대시 실패 시 배터리를 소모하지 않는다.

### 1-5. 대시 잔상 이펙트

대시 동작 시 캐릭터의 잔상 이펙트를 적용한다.

규칙:

- 대시 시작 시 현재 캐릭터 스프라이트 잔상 3~5개를 생성한다.
- 잔상은 `0.12~0.18초` 동안 fade out된다.
- 잔상 위치는 대시 경로를 따라 배치한다.
- 잔상은 충돌 없음 / 입력 없음 / 순수 visual이다.
- 잔상을 실제 플레이어 노드로 만들면 안 된다.
- 잔상은 `Sprite2D` 또는 `AnimatedSprite2D` 복사본처럼 시각 노드로만 처리한다.
- 잔상은 fade out 후 제거한다.

### 1-6. 벽타기 삭제

벽타기 기능은 삭제한다.

삭제 방향:

- 벽타기 입력을 제거한다.
- 벽타기 상태를 더 이상 사용하지 않는다.
- 벽타기 관련 배터리 소모를 제거한다.
- 벽타기 관련 코드도 모두 삭제한다.

주의:

- 벽 방향 입력은 더 이상 벽타기용으로 사용하지 않는다.
- 벽 방향 입력은 방향키 기반 벽 채굴 진입 조건으로 사용한다.

---

## 2. 채굴 시스템 변경

### 2-1. 채굴 입력 변경

채굴 동작 방식을 기존 우클릭에서 방향키 기반 채굴로 변경한다.

규칙:

- 더 이상 우클릭이 채굴을 동작시키지 않는다.
- 우클릭은 대시 전용 입력으로 사용한다.
- 채굴은 좌/우 벽과 접촉한 상태에서 벽 방향 입력을 유지하는 방식으로 동작한다.

### 2-2. 채굴 상태

채굴 관련 상태는 `is_digging` 상태로 구분한다.

규칙:

- `is_digging == true`일 때만 실제 벽 채굴 상태로 본다.
- 드릴 관련 이미지는 `is_digging` 상태에서만 표기한다.
- `is_digging == false`일 때는 드릴 이미지를 표시하지 않는다.

---

## 3. is_digging 조건

### 3-1. is_digging 진입 조건

아래 조건을 모두 만족하면 `is_digging` 상태로 진입한다.

- 공격 중이 아님
- 대시 중이 아님
- 채굴 가능한 좌/우 벽과 접촉 중
- 접촉한 벽 방향 입력을 `0.45초` 이상 유지
- 배터리가 충분함
- intermission 채굴 잠금 상태가 아님

### 3-2. is_digging 해제 조건

아래 조건 중 하나라도 만족하면 `is_digging` 상태를 해제한다.

- 공격키 입력
- 방향키 해제
- 반대 방향 입력
- 점프 입력
- 대시 시작
- 배터리 부족
- 채굴 가능 대상 없음 + 연속채굴 유예시간 종료

### 3-3. is_digging 유지 조건

`is_digging` 상태에서 채굴 방향키를 유지하면 `is_digging` 상태도 유지한다.

규칙:

- 같은 방향 입력을 유지하는 동안 채굴 상태를 유지한다.
- 공격, 점프, 대시, 반대 방향 입력이 발생하면 즉시 해제한다.
- 배터리가 부족해지면 즉시 해제한다.

---

## 4. 연속채굴 처리

`is_digging` 상태에서 벽 채굴 대상이 파괴되고 캐릭터가 다음 채굴 대상을 향해 이어서 채굴하려는 상황에서는 잠시 동안 `is_digging` 상태를 유지한다.

규칙:

- 연속채굴 유예시간은 `0.45초`다.
- 유예시간 안에 다음 채굴 가능 대상이 잡히면 `is_digging` 상태를 유지한다.
- 유예시간 안에 다음 채굴 가능 대상이 잡히지 않으면 `is_digging` 상태를 해제한다.

의도:

- 벽을 연속으로 채굴할 때 드릴 이미지가 깜빡이지 않도록 한다.
- 벽 채굴 대상 하나가 파괴될 때마다 채굴 타이머와 연출이 불필요하게 끊기지 않도록 한다.

---

## 5. 채굴 진입 대기 처리

`is_digging` 진입 대기 중에도, 채굴 가능한 벽과 접촉하고 접촉 방향 입력을 유지 중이라면 플레이어가 벽에서 과도하게 미끄러져 내려가지 않도록 vertical velocity를 약하게 제한한다.

규칙:

- 이 상태는 채굴 진입 대기 상태일 뿐이다.
- 이 상태에서는 `dig_effect`가 발생하지 않는다.
- 이 상태에서는 `dig_execute`가 발생하지 않는다.
- 접촉 방향 입력이 `0.45초` 이상 유지되면 `is_digging` 상태로 진입한다.
- 접촉 방향 입력이 끊기면 채굴 진입 대기 상태를 취소한다.

의도:

- 높은 층수의 벽에 대시로 접근한 뒤 채굴 진입 전 0.45초 동안 바로 떨어지는 문제를 완화한다.
- 벽타기 삭제 이후에도 높은 위치의 벽을 안정적으로 채굴할 수 있게 한다.

---

## 6. dig_effect 이벤트

`is_digging` 상태에서 채굴속도에 비례해 n초마다 `dig_effect` 이벤트가 발생한다.

기본 주기:

```text
BASE_DIG_EFFECT_INTERVAL = 0.5초
```

규칙:

- `dig_effect` 이벤트 1회당 캐릭터가 앞뒤로 드릴의 반동을 느끼는 듯한 가벼운 진동 이펙트 또는 애니메이션을 적용한다.
- `dig_effect`는 체감 피드백 전용이다.
- `dig_effect`는 실제 채굴 데미지를 가하지 않는다.
- `dig_effect`는 배터리를 소모하지 않는다.

---

## 7. dig_execute 이벤트

`is_digging` 상태에서 채굴속도에 비례해 n초마다 `dig_execute` 이벤트가 발생한다.

기본 주기:

```text
BASE_DIG_EXECUTE_INTERVAL = 3.0초
```

규칙:

- `dig_execute` 이벤트 1회당 캐릭터의 채굴데미지만큼 채굴 가능한 벽 서브셀 또는 벽 채굴 대상에 채굴데미지를 가한다.
- `dig_execute` 이벤트 1회당 배터리를 `5` 소모한다.
- 이때 해당 벽 채굴 대상에서 파편 비산 이펙트가 실행된다.
- 파편 비산 이펙트는 이미 구현된 방식을 재사용한다.
- 낙하 블록, 활성 블록, 보스 블록은 채굴 대상이 아니다.

---

## 8. 채굴 데미지 스탯

채굴 데미지 스탯은 `dig_execute` 1회당 벽 채굴 대상에 가하는 데미지를 의미한다.

규칙:

- 채굴 데미지는 고정 수치 증가 방식으로만 상승한다.
- 예: `+1`, `+3`
- 채굴 데미지는 `dig_effect`에는 영향을 주지 않는다.
- 채굴 데미지는 `dig_execute` 1회당 적용되는 피해량에만 영향을 준다.

---

## 9. 채굴 속도 스탯

채굴 속도 스탯은 `is_digging` 상태에서 `dig_effect`와 `dig_execute`가 발생하는 주기를 감소시킨다.

공식:

```gdscript
dig_effect_interval = max(0.12, 0.5 / (1.0 + mining_speed_bonus_percent / 100.0))
dig_execute_interval = max(0.75, 3.0 / (1.0 + mining_speed_bonus_percent / 100.0))
```

### 9-1. 채굴속도 실질 상한

- `dig_execute` 기준 실질 상한은 채굴속도 `+300%`다.
- `dig_execute_interval`은 채굴속도 `+300%`에서 최소 주기 `0.75초`에 도달한다.
- `dig_effect_interval`은 약 `+316.7%`부터 최소 주기 `0.12초`에 도달한다.
- UI에서는 실제 채굴 성능 기준으로 채굴속도 `+300%`를 유효 상한으로 표시할 수 있다.

## 10. is_digging 중 플레이어 상태

`is_digging` 중에는 아래 특징을 지닌다.

규칙:

- 중력 영향을 받지 않거나 매우 약하게 받는다.
- 플레이어의 vertical velocity를 0에 가깝게 보정한다.
- 좌우 이동은 제한한다.
- 반대 방향 입력, 점프, 대시, 공격 입력 시 즉시 해제된다.
- 배터리가 부족하면 즉시 해제된다.

의도:

- 벽타기 삭제 이후에도 높은 위치의 벽을 안정적으로 채굴할 수 있게 한다.
- `is_digging`을 이동 기능이 아니라 벽 채굴 작업 상태로 만든다.

---

## 11. 모래 채굴 삭제

모래 채굴은 삭제한다.

규칙:

- 기존 우클릭 기반 모래 채굴은 더 이상 사용하지 않는다.
- 방향키 기반 `is_digging`은 벽 채굴 전용 상태로 사용한다.
- 모래 셀은 이번 변경 범위에서 채굴 대상이 아니다.
- 모래 채굴은 추후 다른 방법으로 기능을 추가하여 반영할 예정이다.

의도:

- 벽 채굴과 모래 제거를 같은 입력 / 같은 상태로 묶지 않는다.
- 벽 채굴은 탐사 / 보상 루프로 분리한다.
- 모래 제거는 생존 / 탈출 루프로 추후 별도 재설계한다.

---

## 12. 구현 주의사항

- 우클릭은 대시 전용이어야 한다.
- 우클릭으로 채굴이 발동하면 안 된다.
- 기존 대시 입력은 모두 제거해야 한다.
- 벽타기 관련 입력, 상태, 배터리 소모, 코드 흐름은 제거해야 한다.
- `is_digging`은 벽 채굴 전용 상태다.
- 낙하 블록, 활성 블록, 보스 블록은 채굴 대상이 아니다.
- 모래 셀은 이번 변경 범위에서 채굴 대상이 아니다.
- 드릴 이미지는 `is_digging` 상태에서만 표시해야 한다.
- 대시 잔상은 실제 플레이어 노드가 아니라 순수 visual 노드로 생성해야 한다.
