# Burial Protocol - Core Loop Specification

## 0. 목적

이 문서는 Burial Protocol 한 판의 실제 플레이 흐름을 정리한다.
개별 시스템 설명이 아니라, 플레이어가 어떤 입력과 판단을 반복하며
어떤 결과가 루프를 더 유리하거나 불리하게 만드는지를 상위 수준에서 정의한다.

---

## 1. 한 줄 정의

Burial Protocol의 현재 코어 루프는

**타이틀에서 메인 허브로 진입한 뒤, 선택한 캐릭터와 난이도로 런을 시작하고, 좌클릭 공격과 우클릭 채굴로 낙하 블록과 누적 모래를 관리하다가 결과 화면으로 복귀하는 최소 메타-인게임 반복 루프**

로 요약할 수 있다.

---

## 2. 현재 구현 스냅샷

현재 플레이 기준의 고정 전제는 아래와 같다.

- 게임은 타이틀 화면에서 시작한다
- 타이틀에서 메인 허브로 들어간다
- 메인 허브에는 영구 재화 영역, 게임 시작, 업적, 캐릭터 선택, 성장, 아이템 목록 버튼이 있다
- 캐릭터 선택 화면에는 기본 캐릭터 1개와 잠금 placeholder 슬롯이 있다
- 게임 시작은 난이도 선택 팝업을 거쳐 실제 런 씬으로 들어간다
- 블록은 중앙 샤프트 상단에서 주기적으로 생성된다
- 플레이어는 좌우 이동, 점프, 추가 점프, 벽 점프, 좌/우/하 대시가 가능하다
- 좌클릭은 블록 공격, 우클릭은 모래/벽 채굴이다
- 공격은 마우스 방향 기반 4방향 판정이다
- 공격으로 제거한 블록은 골드를 주고 사라진다
- 제거하지 못한 블록은 바닥/모래/압착 상황에서 모래로 분해된다
- 모래는 누적되어 이동 저항과 발판이 된다
- 플레이어는 낙하 블록 위에도 올라탈 수 있고, 블록을 이동 플랫폼처럼 일부 활용할 수 있다
- `R`로 현재 런을 종료하고 결과 화면으로 이동할 수 있다
- `Tab`으로 디버그 라인을 토글할 수 있다
- 체력 0 또는 임시 무게 한계 초과 시 결과 화면으로 이동한다

---

## 3. 바깥 루프와 안쪽 루프

현재 전체 플레이 구조는 바깥 루프와 안쪽 루프로 나뉜다.

### 3-1. 바깥 루프

1. 게임 실행
2. 타이틀 화면 진입
3. 메인 허브 진입
4. 캐릭터 / 성장 / 업적 / 아이템 목록 확인 또는 게임 시작 선택
5. 난이도 선택 팝업
6. 런 시작
7. 결과 화면 진입
8. 메인 허브 복귀

### 3-2. 안쪽 루프

런 안에서는 아래 반복이 핵심이다.

- 낙하 블록을 공격으로 제거할지 판단한다
- 제거 실패로 쌓이는 모래를 관리한다
- 우클릭 채굴로 모래나 벽을 정리한다
- 점프와 대시로 즉시 위협을 회피한다
- 압박이 누적되면 다시 위치를 바꾸고 공간을 연다

---

## 4. 플레이어가 반복하는 핵심 행동

플레이어는 한 판 동안 아래 행동을 계속 반복한다.

- 상단에서 내려오는 블록의 위치와 크기를 확인한다
- 지금 공격으로 제거할 블록을 고른다
- 여러 블록을 한 번에 맞출 수 있는 위치를 잡는다
- 제거하지 못할 블록은 회피하거나 점프로 넘는다
- 필요하면 좌/우/하 대시로 즉시 벗어난다
- 발밑 모래를 밀며 이동한다
- 모래 더미를 발판으로 활용해 높이를 바꾼다
- 우클릭으로 모래를 먼저 채굴하고, 필요하면 벽을 채굴해 빠질 공간을 만든다
- 눌리기 직전 상황을 피하거나, 이미 불리해진 지형을 다시 유리하게 만든다

즉, 플레이어는
단순히 낙하물을 피하는 것이 아니라,
공격, 회피, 채굴, 모래 지형 활용을 동시에 수행한다.

---

## 5. 결과 분기

한 번의 블록 처리 결과는 크게 세 갈래로 나뉜다.

### 5-1. 공격 파괴 성공

플레이어가 블록 체력을 0 이하로 만들면:

- 블록은 즉시 제거된다
- 모래를 남기지 않는다
- 골드가 지급된다
- 데미지 팝업이 뜬다

의미는 아래와 같다.

- 즉시 위협 제거
- 공간 보존
- 자원 획득

### 5-2. 공격 실패 후 낙하 지속

블록 체력이 남아 있으면:

- 블록은 계속 낙하한다
- 이후 다시 공격 대상이 될 수 있다
- 플레이어 압착 또는 분해 조건으로 이어질 수 있다

의미는 아래와 같다.

- 위협이 계속 유지된다
- 다음 판단을 강제한다

### 5-3. 분해

블록이 아래 조건 중 하나를 만족하면 분해된다.

- 바닥에 닿음
- 이미 쌓인 모래에 닿음
- 플레이어를 압착해 피격을 발생시킴

분해되면:

- 블록은 사라진다
- 골드는 지급되지 않는다
- 블록 타입에 맞는 모래가 생성된다

의미는 아래와 같다.

- 즉시 위협은 사라지지만
- 장기적으로 더 나쁜 지형이 남는다

---

## 6. 코어 루프의 두 층

### 6-1. 즉시 위협

즉시 위협은 낙하 블록이다.

플레이어는 아래를 빠르게 판단한다.

- 이 블록을 지금 때릴 수 있는가
- 한 번에 여러 개를 맞출 수 있는가
- 그냥 피하는 편이 나은가
- 점프나 벽 쪽 이동이 더 안전한가

### 6-2. 누적 위협

누적 위협은 분해되어 쌓인 모래다.

플레이어는 아래를 계속 판단한다.

- 이 모래를 밟고 올라설 것인가
- 밀고 지나갈 수 있는가
- 속도 감소를 감수할 가치가 있는가
- 채굴로 옆 공간을 확보하는 편이 나은가

즉, 낙하 블록은 순간 압박이고,
모래는 시간이 갈수록 커지는 환경 압박이다.

---

## 7. 현재 액션 루프

현재 액션은 공격과 채굴로 분리되어 있다.

### 7-1. 공격

1. 좌클릭 입력 발생
2. 플레이어 기준 마우스 방향으로 공격 판정 생성
3. 범위 안의 낙하 블록을 타격한다
4. 블록을 맞추지 못하면 공격 miss 상태 문구만 남긴다

### 7-2. 채굴

1. 우클릭 입력 발생
2. 플레이어 기준 마우스 방향으로 채굴 판정 생성
3. 모래 채굴 시도
4. 겹치는 벽도 있으면 벽 채굴 시도
5. 둘 다 없으면 miss 상태 문구를 남긴다

즉, 현재 코어 루프에서 공격과 채굴은 별도 입력/쿨다운을 가진다.

---

## 8. 공격의 의미

현재 구현에서 공격은 아래 의미를 가진다.

- 가장 빠른 위협 제거 수단
- 골드 획득 수단
- 모래 누적 방지 수단
- 공간 유지 수단

공격에 성공할수록:

- 체력 손실 가능성이 줄고
- 중앙 샤프트가 덜 막히며
- 이동 경로가 깨끗하게 유지된다

---

## 9. 채굴의 의미

채굴은 직접 보상을 주지는 않지만,
현재 구조에서 매우 중요한 생존 보조 수단이다.

채굴의 역할은 아래와 같다.

- 좌우 벽에 탈출구를 만든다
- 모래/블록 압박이 심할 때 옆으로 숨을 공간을 만든다
- 상하 이동만으로는 해결 안 되는 상황을 풀어준다

현재 구현에서는:

- 한 번의 채굴로 모래와 벽에 각각 hit/remove 결과가 날 수 있다
- 잠긴 메타 시스템과 무관하게 런 생존용 행동으로만 쓰인다
- 채굴 범위는 플레이어 충돌 바디 기준으로 정렬된다

---

## 10. 모래의 의미

모래는 단순 장애물이 아니다.
현재 코어 루프에서 모래는 아래 역할을 동시에 가진다.

- 이동 저항
- 지형 누적
- 임시 발판
- 점프와 우회 경로의 일부
- 장기 압박

플레이어는 모래를 완전히 제거할 수 없기 때문에,
모래를 어떻게 밟고, 밀고, 넘느냐가 생존 기술이 된다.

---

## 11. 플레이어의 반복 판단 구조

플레이어는 보통 아래 질문을 반복한다.

### 11-1. 지금 공격이 이득인가

- 맞출 수 있는가
- 여러 블록을 같이 맞출 수 있는가
- 지금 처리하지 않으면 모래가 더 쌓이는가

### 11-2. 지금 점프나 대시가 필요한가

- 머리 위 낙하 블록을 피해야 하는가
- 모래 더미를 넘으려는가
- 벽이나 높은 발판으로 접근하려는가
- 즉시 좌우 또는 아래로 빠져야 하는가

### 11-3. 지금 채굴해야 하는가

- 중앙 공간이 답답한가
- 앞 모래를 바로 비워야 하는가
- 좌우 벽 쪽에 비상 경로가 필요한가
- 공격보다 공간 확보가 더 시급한가

### 11-4. 지금 모래를 활용할 것인가

- 이 모래 위에 올라탈 수 있는가
- 밀어가며 전진할 수 있는가
- 이 위치가 다음 낙하를 피하기 쉬운가

---

## 12. 현재 실패 구조

현재 실패는 단일 한 번의 실수보다,
반복적인 제거 실패가 누적될 때 커진다.

실패 누적 흐름은 대체로 아래와 같다.

1. 블록 제거 실패
2. 블록 분해로 모래 누적
3. 이동 저항 증가
4. 회피 공간 축소
5. 압착 위험 증가
6. 체력 감소
7. 더 많은 블록 처리 실패

주의할 점은 아래와 같다.

- 현재 구현에는 결과 화면 전환이 있다
- 결과 화면은 최소 정보만 보여주고 메인 허브로 복귀시킨다
- 정교한 게임오버 연출과 보상 정산은 아직 없다

---

## 13. 현재 루프가 잘 작동하는 조건

아래 조건이 만족되면 현재 코어 루프가 잘 작동한다고 본다.

- 낙하 블록이 실제 위협으로 느껴진다
- 공격이 즉시 이득으로 체감된다
- 공격 실패가 모래 누적으로 되돌아온다
- 모래가 단순 방해물이 아니라 발판으로도 쓰인다
- 채굴이 실제 탈출 수단으로 의미를 가진다
- 플레이어가 공격, 점프, 채굴, 위치 선정 사이에서 계속 선택을 강요받는다
- 메타 허브가 런 시작 전 상태 관리 화면으로 기능한다

---

## 14. 테스트 기준

- 타이틀에서 메인 허브로 진입 가능해야 한다
- 메인 허브에서 캐릭터 / 성장 / 업적 / 아이템 / 게임 시작 진입이 가능해야 한다
- 난이도 선택 팝업 뒤 런이 시작되어야 한다
- 결과 화면이 최소 정보와 나가기 버튼을 가져야 한다
- 블록이 일정 간격으로 중앙 샤프트 상단에서 계속 생성되어야 한다
- 좌클릭 공격으로 블록을 제거하면 골드가 증가해야 한다
- 우클릭 채굴로 모래와 벽에 반응해야 한다
- 분해된 블록은 모래를 남겨야 한다
- 모래가 누적되면서 이동 감각을 바꿔야 한다
- 모래 더미를 발판으로 활용할 수 있어야 한다
- 좌우 벽 채굴이 실제로 탈출 공간을 만들어야 한다
- 낙하 블록 위에 올라탔을 때 점프 타이밍이 너무 흔들리지 않아야 한다

---

## 15. 결론

Burial Protocol의 현재 코어 루프는
타이틀과 메인 허브를 거쳐 런에 들어가,
낙하 블록을 공격으로 정리해 공간과 골드를 확보하고,
실패로 누적되는 모래를 다시 지형으로 활용하며,
막힐 때는 모래와 좌우 벽을 파서 생존 공간을 회복한 뒤,
결과 화면을 통해 다시 허브로 돌아오는 반복 구조다.

즉, 이 게임의 핵심은
"즉시 위협 제거"와 "실패 결과로 남은 환경 압박 관리"를
동시에 요구하는 생존 루프를
최소 메타 허브 루프 안에 묶는 데 있다.
