# META LOOP MIN SPEC

이 문서는 Burial-Protocol의 **최소형 전체 루프 / 메타 루프**에 대한 구현 기준 문서이다.  
이번 범위는 **타이틀 → 메인 허브 → 하위 메뉴 → 런 시작 → 결과 화면**까지의 전체 흐름을 정의한다.

이 문서는 구현 범위를 명확히 고정하기 위한 문서이며, 구현자는 임의로 범위를 확장하지 않는다.

---

## 1. 문서 목적

이 문서의 목적은 아래 항목들을 최소 구현 범위 기준으로 확정하는 것이다.

- 타이틀 화면 역할
- 메인 허브 화면 역할
- 캐릭터 목록 화면 역할
- 게임 시작 전 흐름
- 난이도 선택 방식
- 결과 화면의 최소 표시 정보
- 저장 시점
- 영구 재화 개념
- 현재 구현 범위에서 제외할 항목

---

## 2. 전체 루프 구조

최소형 전체 루프는 아래 구조를 따른다.

1. 게임 실행
2. 타이틀 화면 진입
3. 타이틀에서 시작 진입
4. 메인 허브 화면 진입
5. 메인에서 캐릭터 / 성장 / 업적 / 아이템 목록 / 게임 시작 진입
6. 게임 시작 클릭
7. 난이도 선택 팝업 표시
8. 런 시작
9. 런 종료
10. 결과 화면 표시
11. 나가기 버튼으로 메인 또는 타이틀 복귀

---

## 3. 화면 구조

### 3.1 타이틀 화면

타이틀 화면은 **연출 + 시작 진입 화면**이다.  
실제 허브 기능은 메인 화면이 담당한다.

#### 구성

- 배경: 그래픽 이미지
- 상단: 게임 제목 UI
- 중앙보다 약간 아래부터 버튼 목록 배치

#### 버튼 목록

- 게임 시작
- 설정
- 프로필
- 게임 종료
- 랭킹

#### 역할

- 게임 최초 진입 시 보여주는 연출 화면
- 메인 허브로 진입하기 위한 입구 역할
- 실제 영구 재화 관리 / 캐릭터 선택 / 성장 관리 기능은 담당하지 않음

---

### 3.2 메인 화면

메인 화면은 **실제 허브 화면**이다.  
전체 루프 관리와 주요 메뉴 진입 기능을 담당한다.

#### 구성

- 영구성 재화 목록 UI
- 게임 시작 버튼
- 업적 버튼
- 캐릭터 선택 버튼
- 성장 버튼
- 아이템 목록 버튼
- 화면 중앙: 현재 선택된 캐릭터 표시 UI
- 캐릭터 표시 UI 아래: 해당 캐릭터의 최고기록 표시

#### 역할

- 현재 프로필 상태를 요약해서 보여주는 화면
- 실제 게임 시작 전 상태 관리 허브
- 캐릭터 / 성장 / 업적 / 아이템 목록 화면 진입점

---

### 3.3 캐릭터 목록 화면

#### 구성

- 전체 캐릭터 목록 표시
- 해금 조건 확인 가능
- 캐릭터별 최고기록 간략 표기

#### 캐릭터 선택 시스템

- 초기에는 1개의 기본 캐릭터만 존재
- 최소 10종 이상의 캐릭터를 추후 구현 예정
- 현재는 모두 비공개 잠금 상태
- 잠금 캐릭터는 hover 시 해금 조건을 간단히 보여주는 툴팁 표시
- 캐릭터별 능력 차이가 있을 예정

---

### 3.4 업적 화면

- 추후 구현
- 이번 최소 구현 범위에서는 실제 업적 기능 구현 제외

---

### 3.5 성장 화면

- 추후 구현
- 영구 성장 관련 항목용 화면
- 이번 최소 구현 범위에서는 실제 성장 트리 구현 제외

---

### 3.6 아이템 목록 화면

- 도감 / 보유 목록 화면 성격
- 이번 최소 구현 범위에서는 실제 상세 시스템 구현보다 화면 진입 구조 정의가 우선

---

### 3.7 결과 화면

#### 최소 표시 항목

- 이번 게임의 기록
- 나가기 버튼

#### 비고

- 현재 확정 기준은 위 두 항목이다
- 재시작 버튼, 메인으로 버튼 등은 추후 확장 가능하지만 이번 문서의 확정 범위는 아님

---

## 4. 화면 간 이동 규칙

### 4.1 메인 화면 기준 이동

- 게임 시작 → 난이도 선택 팝업 → 런 시작
- 캐릭터 선택 → 캐릭터 목록 화면
- 성장 → 성장 화면
- 업적 → 업적 화면
- 아이템 목록 → 도감 / 보유 목록 화면

### 4.2 게임 시작 전 결정 순서

1. 메인 화면에서 캐릭터는 미리 선택되어 있음
2. 게임 시작 클릭
3. 난이도 선택 팝업 표시
4. 시작

---

## 5. 스테이지 구조

- 한 게임은 2~30 스테이지로 구성
- 스테이지마다 일정 시간 소요
- 최종 스테이지까지 버틸 경우 보스 등장

### 클리어 조건

아래 둘 중 하나를 만족하면 클리어 처리한다.

1. 보스 블록을 처치
2. 보스 블록을 처치하지 못하고 모래화되었더라도, 게임오버 조건에 충족되지 않은 상태라면 클리어 처리  
   (최대 무게 초과 없음, 체력 0 이하 아님)

### 난이도

- 노말
- 하드
- 헬
- 익스트림

### 최고기록의 의미

- 어떤 난이도에서 몇 스테이지까지 도달했는지

---

## 6. 게임오버 조건

아래 중 하나를 만족하면 게임오버 처리한다.

1. 현재 체력이 0이 될 경우
2. 깔린 모래의 수만큼 산정된 무게가 해당 게임의 최대 무게를 초과하고, 그 결과 바닥이 무너질 경우

게임오버 이후에는 결과 화면을 표시한다.

---

## 7. 게임오버 연출

### 7.1 무게 초과 시

- 바닥이 흔들림
- 바닥이 무너짐
- 캐릭터 포함 모든 것이 아래로 떨어짐
- 이후 결과창 표시

### 7.2 체력 소진 시

- 화면에 노이즈 발생
- 화면이 어두워짐
- 이후 결과창 표시

---

## 8. 영구 성장 관련 재화

### 재화 종류

- 기어: 공격 스탯 강화, 채굴 스탯 강화
- 합판: 방어 스탯 강화
- 윤활유: 기동성 스탯 강화
- 철광석: 공통 재료
- 전력: 공통 재료

### 획득 규칙

- 철광석과 전력은 게임 클리어 시 획득하는 재화
- 처리한 블록 등에 비례하여 지급
- 기어 / 합판 / 윤활유는 보물상자 등으로 소량 획득

### 제작 개념

- 철광석과 전력으로 기어 / 합판 / 윤활유 제작 가능
- 현재는 **제작 개념만 저장**
- 실제 제작 UI 구현은 이번 범위에서 보류

---

## 9. 저장 시점

아래 시점에 저장한다.

- 런 종료 시
- 영구 성장 시
- 설정 변경 시
- 업적 획득 시
- 캐릭터 해금 시

---

## 10. 프로필 구조

- 이 게임은 **단일 프로필 방식**
- 멀티 슬롯 개념은 존재하지 않음

---

## 11. 현재 최소 구현 범위에서 제외할 것

아래 항목은 이번 범위에서 제외한다.

- 온라인 랭킹
- 실제 업적 구현
- 실제 성장 트리 구현
- 실제 제작 UI 구현
- 다수 캐릭터 밸런싱
- 연출이 풍부한 로비 화면
- 상세 프로필 통계

---

## 12. 구현 범위 규칙

- 임시 UI는 별도의 지시를 받은 뒤에만 제작 돌입
- 이번 문서는 **최소형 전체 루프 구현 기준**을 고정하기 위한 문서
- 구현자는 임의로 업적 / 성장 / 제작 / 다캐릭터 밸런싱 범위를 확장하지 않음

---

## 13. 요약

### 확정된 것

- 타이틀 = 연출 + 시작 진입
- 메인 = 실제 허브
- 게임 시작 전 흐름 = 메인에서 캐릭터 미리 선택 → 게임 시작 → 난이도 선택 팝업 → 시작
- 최고기록 = 난이도별 도달 스테이지
- 결과 화면 최소 정보 = 이번 기록 + 나가기 버튼
- 저장 시점 명확히 확정
- 단일 프로필 방식 확정

### 추후 구현

- 업적 실제 기능
- 성장 트리 실제 기능
- 제작 UI
- 다수 캐릭터 실제 밸런싱
- 상세 프로필 및 고급 통계

### 이번 범위 핵심

- 전체 루프의 뼈대를 먼저 만든다
- 각 화면의 역할과 이동 흐름을 먼저 고정한다
- 상세 콘텐츠 구현은 뒤로 미룬다
