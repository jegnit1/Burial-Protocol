# Burial Protocol - Gameplay Systems Specification

## 0. 목적

이 문서는 현재 코드 기준 실제 동작하는 게임플레이 시스템을 정리한다.
입력, 이동, 전투, 채굴, 블록 데이터 모델, 모래, Day 전환, 스탯, HUD까지 포함한다.

기준일: `2026-04-20`

---

## 1. 입력 체계

현재 기본 입력:

- 좌 이동: `A`, `Left Arrow`
- 우 이동: `D`, `Right Arrow`
- 점프: `W`, `Up Arrow`, `Space`
- 아래 입력/급강하: `S`, `Down Arrow`
- 공격: `Left Mouse Button`
- 채굴: `Right Mouse Button`
- 대시 액션: `Z`
- 상호작용: `E`
- 디버그 패널 토글: `Tab`
- 일시정지 메뉴: `ESC`
- 수동 종료/리스타트: `R`

입력 보장은 `GameConstants.ensure_input_actions()`가 담당한다.

---

## 2. 플레이어 이동

### 2-1. 기본 수치

- 지상 이동 속도: `426 px/s`
- 공중 이동 속도: `373 px/s`
- 모래 속 이동 배율: `0.62`
- 중력: `2000 px/s^2`
- 기본 점프 속도: `-853`
- 추가 점프: `1회`
- 벽 점프 가로 속도: `480`
- 점프 버퍼: `0.14초`
- 코요테 타임: `0.10초`
- 빠른 낙하 가속: `2933 px/s^2`
- 빠른 낙하 최대 속도: `1306 px/s`

### 2-2. 대시

현재 대시는 아래 두 방식이 모두 동작한다.

1. 좌/우/하 더블탭
2. 방향 입력 중 `Z`

지원 방향:

- 좌
- 우
- 하

미지원:

- 상향 대시

대시 수치:

- 거리: `4U`
- 지속시간: `0.08초`
- 더블탭 입력창: `0.22초`
- 쿨다운: `0.45초`

### 2-3. 벽타기

기존 우발적 벽 슬라이드는 정식 벽타기 시스템으로 교체되었다.

발동 조건:

- 공중 상태
- 고정벽 접촉
- 접촉한 벽 방향 입력 유지
- 배터리 `> 0`
- 대시 중 아님

특징:

- 낙하 속도 제한: `110 px/s`
- 벽타기 중 배터리 지속 소모
- 벽타기 중 점프 횟수 제한 무시
- 점프 후 재진입 쿨타임 없음

배터리 수치:

- 최대 배터리: `100`
- 소모량: `5 / sec`
- 회복량: `5 / sec`

현재 회복 규칙:

- 벽타기 상태일 때만 회복 불가
- 그 외 상태에서는 공중이든 지상이든 회복

중요:

- 벽타기는 `WorldGrid`의 고정벽 충돌만 사용한다
- falling block 측면, 모래, 동적 오브젝트에는 벽타기 불가

## 3. 공격 시스템

### 3-1. 기본 수치

- 기본 공격력: `10`
- 기본 공격 쿨다운: `0.25초`
- 공격 버퍼: `0.12초`
- 공격 미리보기 지속: `0.12초`
- 기본 공격 범위: 전방 `1U x 1U`

### 3-2. 판정 방식

- 마우스 방향 기반
- 너무 가까우면 현재 facing 방향 사용
- 회전 가능한 직사각형 shape query 사용
- 활성 낙하 블록만 공격 대상

### 3-3. 치명타

현재 공격에만 치명타가 적용된다.
채굴에는 적용되지 않는다.

- 기본 치명타 확률: `1%`
- 치명타 배율: `200%`

치명타 발생 시:

- 최종 공격 피해 2배 적용
- 데미지 팝업 노란색
- 숫자 뒤에 `!` 표시

---

## 4. 플레이어 피격 시스템

### 4-1. 기본 체력

- 최대 체력: `100`

### 4-2. 압착 피해

낙하 블록이 플레이어를 압착할 때 피해는 고정 1이 아니다.
현재 공식:

`블록 면적 U 수 x 10`

예:

- `1x1` 블록 = `10`
- `1x2` 블록 = `20`
- `2x1` 블록 = `20`

여기에 방어력이 적용된 뒤 최소 1 피해를 받는다.

### 4-3. 시각 피드백

플레이어가 피해를 받으면:

- 빨간색 데미지 팝업
- 맞는 순간 붉은 점멸
- 이후 무적 시간 동안 흰색 점멸

무적시간:

- `0.5초`

---

## 5. 체력 재생과 방어

### 5-1. 체력 재생

현재 체력 재생은 일정 주기마다 `1` 회복 방식이다.

재생 간격 공식:

`5 / (1 + (HP 재생 - 1) / 2.25)` 초당 1 회복

현재 기본 구성:

- 기본 HP 재생: `0`
- 테스트 보너스: `10`

즉, 현재 빌드에서는 기본적으로 테스트용 재생이 켜져 있다.
나중에 `PLAYER_TEST_HP_REGEN_BONUS`만 0으로 되돌리면 원래 의도값으로 복귀 가능하다.

### 5-2. 회복 정지 조건

현재 회복은 아래 상태에서만 막힌다.

- 상점 UI 열림
- Next Day 전환 중
- 플레이어 사망
- 이미 최대 체력

### 5-3. 방어력

피해 공식은 현재 아래처럼 적용된다.

`max(원시 피해 - 방어력, 1)`

즉, 방어력은 받는 피해를 줄이지만 최소 1은 보장된다.

---

## 6. 채굴 시스템

### 6-1. 기본 수치

- 기본 채굴 데미지: `1`
- 기본 채굴 쿨다운: `0.15초`
- 채굴 버퍼: `0.12초`
- 기본 채굴 범위: 전방 `0.25U x 1U`

### 6-2. 대상

현재 채굴 대상:

- 모래 셀
- 좌우 벽 서브셀

현재 비대상:

- 낙하 블록

### 6-3. 경험치

- 모래 제거 XP 지급
- 벽 채굴은 XP 없음

### 6-4. intermission 잠금 규칙

Day 종료 후 `3초` 유예가 지나면 채굴만 정지된다.

유지되는 것:

- 플레이어 이동
- 자연스러운 모래 물리 반응
- 플레이어의 모래 밀림

막히는 것:

- 채굴 입력 처리
- 채굴 결과
- 추가 파밍성 제거

---

## 7. 블록 시스템

### 7-1. 데이터 소유권

블록 콘텐츠는 `BlockCatalog.tres`가 소유한다.

- Block Base = 실제 블록 본체
- Block Type = optional affix

스폰 흐름:

1. Base 선택
2. Type 선택 또는 없음
3. Day/Difficulty와 합쳐 `BlockData` 생성

### 7-2. Base가 가지는 값

Base는 실제 블록 본체이므로 아래를 가진다.

- id
- 표시 이름
- 크기
- 기본 HP 관련 값
- 보상
- sand units
- 색상
- 기본 스폰 가중치
- 본체 특수 결과

### 7-3. Type이 가지는 값

Type은 modifier 역할만 담당한다.

- id
- 표시 이름
- HP 배율
- 보상 배율
- sand units 배율/보정
- 스폰 가중치 배율
- 접두/접미 표현
- 추가 특수 결과

### 7-4. 최종 HP 공식

현재 최종 HP는 아래 개념을 따른다.

`1U 기본 HP x 면적 U 수 x Base HP 배율 x Type HP 배율 x Day HP 배율 x Difficulty HP 배율`

즉, 본체 체력의 근본 기준은 Base다.

### 7-5. 현재 블록 종료 방식

활성 낙하 블록은 아래 방식으로 종료된다.

1. 공격으로 파괴
2. 지형/모래에 닿아 분해
3. 플레이어 압착 시 분해

파괴 시:

- 골드 지급
- XP 지급

분해 시:

- `sand_units`만큼 모래 생성
- 골드는 없음

---

## 8. 모래 시스템

### 8-1. 표현

- `1U = 6 x 6` 모래 셀
- 셀 단위 active 영역 시뮬레이션

### 8-2. 이동 규칙

- 아래로 낙하 우선
- 대각선 확산
- 플레이어 이동 중 밀림 해결
- 점프 공간 확보 시도

### 8-3. 중량

실제 실패 기준:

- 모래 셀 한도: `2400`

표시 기준:

- 셀 1개 = `0.1 KG`
- HUD 표시 최대 중량 = `240.0 KG`

즉, 내부 판정 단위와 UI 표시 단위가 분리되어 있다.

---

## 9. Day / intermission / 상점 루프

### 9-1. Day 종료

Day 1~29에서 시간이 끝나면:

1. 일반 스폰 중단
2. intermission 진입
3. 마지막 활성 블록 정리 대기
4. `1.25초` 뒤 키오스크 낙하
5. intermission 시작 후 `3.0초` 뒤 채굴 잠금

### 9-2. 키오스크

- 중앙 상단 낙하
- 속도 `780 px/s`
- 플레이어 비충돌
- 안착 후 상호작용 가능

### 9-3. 상점 단계

현재 `DayShopUI`는 최소 구현 상태다.

- 상점 진입 가능
- `Close`
- `Next Day`

### 9-4. Next Day 전환

`Next Day` 버튼 시:

1. 페이드 아웃
2. 소지 골드의 `10%` 이자 지급
3. 조건부 벽 초기화 훅
4. 새 Day 시작
5. 페이드 인

현재 기본 규칙:

- 벽은 초기화하지 않음
- 모래도 그대로 유지

## 10. 경험치와 레벨업

### 10-1. XP 획득

- 블록 파괴 XP = `width_cells x height_cells x 2`
- 모래 제거 XP = `removed_sand_count x 1`

### 10-2. 레벨업 구조

- 레벨업 준비 시 게임 일시정지
- `LevelUpUI` 팝업
- 카드 3장 제시
- 선택 즉시 적용

### 10-3. 현재 카드 풀

기본 카드:

- 공격력 증가
- 공격속도 증가
- 최대 체력 증가
- 이동속도 증가
- 채굴 데미지 증가
- 채굴 속도 증가

추가 카드:

- 공격범위 증가
- 치명타 확률 증가
- 방어력 증가
- HP 재생 증가
- 점프력 증가
- 채굴범위 증가

현재 제외 항목:

- 치명타 배율: 고정 `200%`
- 이자율: 고정 `10%`
- 행운: 표시만 존재, 효과 없음

---

## 11. ESC 스탯 패널

현재 ESC는 토글형 일시정지 메뉴다.

- 닫힘 -> `ESC` -> 열림
- 열림 -> `ESC` -> 닫힘

우측 패널에는 아래 최종 스탯이 표시된다.

- 공격력
- 공격속도(`/sec`)
- 공격범위
- 치명타 확률
- 치명타 배율
- 현재 체력 / 최대 체력
- 방어력
- HP 재생
- 이동속도
- 점프력
- 채굴 데미지
- 채굴 속도(`/sec`)
- 채굴 범위
- 이자율
- 행운(효과 없음)

공격속도와 채굴속도는 내부 쿨다운이 아니라 표시용 환산값이다.
