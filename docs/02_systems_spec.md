## 0. 목적

이 문서는 Burial Protocol의 실제 구현 기준 시스템 스펙을 하나로 통합한 문서다.
기존의 `game_structure_spec`, `gameplay_systems_spec`, `attack_module_system_spec`, `block material/size draft`, `HUD/UI 문서`에 흩어져 있던 내용을 이 문서로 통합한다.

이 문서는 Codex 또는 개발자가 기능을 수정할 때 우선 참고하는 메인 시스템 스펙이다.

- 현재 legacy 공격모듈 트리거 처리

## 6. Mining

채굴은 우클릭 유지와 마우스 방향으로 처리되는 환경 상호작용이다.

대상:

- 좌우 벽 서브셀

비대상:

- 낙하 블록
- 활성 블록
- 보스 블록

수치:

| 항목 | 값 |
|---|---:|
| 기본 채굴 데미지 | `1` |
| 채굴 기본 주기 | `0.90초` |
| 채굴 최소 주기 | `0.30초` |
| 채굴 배터리 소모 | `5` |
| 채굴 거리 | `0.5U` |
| 채굴 높이 | `0.5U` |

진입 조건:

- 대시 중이 아님
- 마우스 방향에 채굴 가능한 좌우 벽이 있음
- 배터리가 채굴 1회 비용 이상 있음
- 상점 UI 또는 Next Day 전환으로 채굴이 정지된 상태가 아님

채굴 1회는 좌우 벽 채굴 데미지와 파편 처리 경로만 검사하고 배터리 `5`를 소모한다.
채굴 데미지, 채굴속도, 채굴범위는 벽 채굴 전용 스탯이다.

intermission 진입 후에도 벽 채굴과 좌클릭 무기 기반 모래 제거는 유지된다.
플레이어 이동, 공격, 모래 자연 반응, 모래 밀림도 유지한다.
대신 마지막 활성 블록 정리 후 키오스크가 투하되면 중앙 상단 유해물질 HUD가 `10초` 카운트다운을 표시하고, 이후 단계별 고정 환경 피해를 적용한다.

### 6-1. 채굴 확장: 벽 내부 특수 객체

채굴 시스템은 향후 좌우 벽 내부의 특수 객체를 포함하도록 확장한다.

벽 내부 특수 객체는 크게 아래 두 계열로 나눈다.

| 계열 | 의미 | 처리 방식 |
|---|---|---|
| Treasure Chest | 보상형 특수 객체 | 채굴로 노출 후 `E` 상호작용 |
| Creep | 리스크형 특수 객체 | 노출 후 공격 기능으로 파괴 |

의도:

- 채굴을 단순 벽 제거가 아니라 탐사/보상/리스크 관리 루프로 확장한다.
- 중앙 전투와 좌우 벽 채굴의 역할을 분리한다.
- 채굴 빌드가 보상 탐색과 위험 대응에 의미를 가지도록 한다.

### 6-2. 보물상자 벽블록

런 중 좌우 벽 내부에는 간헐적으로 빛나는 이펙트가 표시되는 벽블록이 존재할 수 있다.

규칙:

- 빛나는 벽블록은 보물상자가 숨겨진 후보임을 의미한다.
- 해당 벽블록을 채굴하면 보물상자가 등장한다.
- 보물상자는 즉시 획득되지 않고, 월드 오브젝트로 노출된다.
- 플레이어가 보물상자 근처에서 `E` 상호작용을 하면 획득 팝업이 열린다.
- 보상 팝업이 열린 동안 게임은 일시정지된다.
- 보상 팝업에는 랜덤 보상 연출을 적용한다.

보물상자 자체 등급:

- Normal
- Silver
- Gold
- Platinum

보물상자 등급별로 아래 요소가 달라진다.

- 보물상자 이미지
- 빛나는 벽블록 이펙트 색상/강도
- 보상 테이블
- 고등급 보상 등장 확률
- 보상 저점
- 등장 위치 경향성

고등급 보물상자는 아래 경향성을 가진다.

- 더 깊은 벽 안쪽에 등장할 확률이 높다.
- 더 높은 위치의 벽에서 등장할 확률이 높다.
- 초반/저난이도에서는 출현율을 낮게 제한한다.

### 6-3. 보물상자 보상 체계

보물상자의 모든 보상은 아이템 체계와 동일하게 `D~S` 등급을 가진 보상 항목으로 정의한다.

보상은 `소량/보통/대량` 같은 표현을 사용하지 않는다.
대신 아래처럼 등급이 붙은 보상 아이템 단위로 관리한다.

예시:

- `S급 골드주머니`
- `B급 경험치 물약`
- `C급 모래제거 프로토콜`
- `A급 공격모듈 보급권`
- `D급 강화모듈 부품`

보상 대분류:

| 분류 | 예시 | 설명 |
|---|---|---|
| Gold | `D~S급 골드주머니` | 골드 지급 보상 |
| XP | `D~S급 경험치 물약` | 경험치 지급 보상 |
| Item(Module) | `D~S급 모듈 보급권`, `D~S급 강화모듈 부품` | 공격/기능/강화 모듈 계열 보상 |
| Sand Removal | `D~S급 모래제거 프로토콜` | 일정량의 모래를 제거하는 보상 |

보물상자 등급과 보상 등급은 별도 축이다.

예:

- Silver 보물상자에서 C~A급 보상 중심으로 등장
- Gold 보물상자에서 B~S급 보상 중심으로 등장
- Platinum 보물상자에서 A~S급 보상 중심으로 등장

원칙:

- 보물상자 등급별 보상 테이블은 서로 다르게 정의한다.
- 보상의 세부 수치와 확률은 추후 데이터 테이블에서 설계한다.
- 채굴은 블록 처리를 포기하고 수행되는 선택이므로 보상 저점을 어느 정도 보장한다.
- 고등급 보물상자에서 기대치를 크게 낮추는 보상은 제한한다.
- 일정 등급 이상의 보물상자는 D/C급 모듈을 제공하지 않거나, 낮은 등급 보상이 나올 경우 최소 B~S급 XP/Gold/Sand Removal 보상으로 보정한다.
- 모래삭제 보상은 레벨업 카드가 아니라 보물상자 보상으로만 사용 가능하다.

### 6-4. 크립

벽 내부에는 보물상자뿐 아니라 리스크형 특수 객체도 존재한다.
이를 개발 스펙상 `Creep`으로 정의한다.

크립 규칙:

- 크립은 좌우 벽 내부에 숨겨진 위협 오브젝트다.
- 크립은 채굴로 노출될 수 있다.
- 크립은 채굴로 파괴하지 않고 공격 기능으로 파괴한다.
- 크립을 파괴하지 않으면 지속적으로 플레이어를 방해한다.
- 크립은 노출 즉시 과도한 피해를 주지 않고, 짧은 경고/활성화 시간을 가진다.

크립 분류:

| 분류 | 역할 | 예시 |
|---|---|---|
| 공격형 | 전투 영역에 주기적으로 공격 | 벽속에서 가시를 전투영역 방향으로 발사 |
| 디버프형 | 플레이어에게 지속 디버프 | 획득 재화 감소, 행운 감소, 점프력 감소, 공격력 감소 |

공격형 크립:

- 일정 주기로 전투영역 방향에 공격을 가한다.
- 예시는 스타크래프트 럴커처럼 벽 안에서 가시를 발사하는 방식이다.
- 공격은 피하거나 대응할 수 있어야 한다.
- 방치하면 블록 처리와 이동을 방해한다.

디버프형 크립:

- 처치하지 않을 경우 플레이어에게 각종 디버프를 건다.
- 예시 디버프는 획득 재화 감소, 행운 감소, 점프력 감소, 공격력 감소다.
- 특성상 저난이도나 저스테이지에서 출현율을 강하게 제한한다.
- 동시 활성 수가 많아지면 불쾌감이 커지므로 활성 개수 제한이 필요하다.

크립의 종류와 세부 효과, 수치, 출현 조건은 추후 별도 데이터 단계에서 상세 정의한다.

## 7. Equipment

### 7-1. 설계 기준

신규 전투 장비는 무기, 드론, 드론 프로토콜, 패시브 모듈로 분리한다.
현재 코드의 `attack_module` 경로는 다음 Phase에서 교체할 legacy 런타임이다.

| 장비 | 슬롯 | 등급 | 발동 |
|---|---:|---|---|
| 무기 | `2개` | 데이터 기준 `D~S` | 좌클릭 시 각 무기가 독립 쿨타임으로 발사 |
| 드론 | `1개` | 현재 없음 | 플레이어 상단에 표시되고 프로토콜 실행 |
| 드론 프로토콜 | `5개` | `D~S` | 자동 행동. 동일 프로토콜 중복 가능 |
| 패시브 모듈 | `5개` | `D~S` | 직접 공격하지 않고 조건부 추가 효과 제공 |

우측 무기는 우클릭으로 발사하지 않는다.
우클릭 채굴 입력은 유지한다.

### 7-2. 속성

```text
electric / fire / physical / energy / chemical / none
```

### 7-3. 유형

```text
support / projectile / area / beam / chain / explosion
```

기존 `melee`, `ranged`, `mechanic`은 신규 유형이 아니다.
legacy 데이터 마이그레이션 시 무기, 드론 프로토콜, 속성, 유형으로 재분류한다.

### 7-4. 전투 스탯

| 스탯 | 적용 대상 |
|---|---|
| 무기 공격력 | 무기 피해 |
| 드론 공격력 | 공격형 드론 프로토콜 피해 |
| 공격속도 | 무기 쿨타임 |
| 드론 쿨타임 감소 | 프로토콜 쿨타임. 레벨업 기본 풀 제외 |

기존 근거리 공격력과 원거리 공격력은 제거 또는 deprecated 처리한다.

### 7-5. 공격 대상

무기 대상:

- 활성 낙하 블록
- 보스 블록
- 모래 셀

공격형 드론 프로토콜 대상:

- 활성 낙하 블록
- 보스 블록

공통 비대상:

- 벽
- 플레이어
- 환경물

드론 프로토콜은 모래 피해를 줄 수 없다.
`sand_cleaner`는 피해가 아닌 별도 특수 제거 효과다.

### 7-6. 치명타

- 기본 치명타 확률: `1%`
- 치명타 배율: `200%`

치명타는 공격에만 적용되며 채굴에는 적용되지 않는다.

## 8. Block / Spawn

### 8-1. 최신 블록 모델

최신 블록 모델:

```text
Runtime Block = Material x Size + optional Type
```

- Material = 재질/성질
- Size = 가로/세로 크기
- Type = 선택 affix/modifier

기존 코드에 `base` 명칭이 남아 있을 수 있으나 문서상 기준은 material/size/type이다.

### 8-2. 스폰 흐름

1. 현재 난이도와 Day 확인
2. `BlockCatalog`에서 유효 candidate 수집
3. candidate weight 기반 랜덤 선택
4. optional type 선택
5. `BlockSpawnResolver`가 resolved definition 생성
6. `BlockData`로 변환
7. `FallingBlock` 생성

### 8-3. 최종 HP

```text
final_hp =
  BLOCK_HP_PER_UNIT
  x size_hp_multiplier
  x material_hp_multiplier
  x difficulty_hp_multiplier
  x type_hp_multiplier
```

기본값:

- `BLOCK_HP_PER_UNIT = 10.0`

### 8-4. 최종 보상

```text
final_reward =
  BLOCK_REWARD_PER_UNIT
  x size_reward_multiplier
  x material_reward_multiplier
  x type_reward_multiplier
```

기본값:

- `BLOCK_REWARD_PER_UNIT = 5.0`

### 8-5. 최종 모래량

```text
final_sand_units =
  BLOCK_SAND_UNITS_PER_UNIT
  x size_reward_multiplier
  x type_sand_units_multiplier
```

기본값:

- `BLOCK_SAND_UNITS_PER_UNIT = 36.0`

### 8-6. 블록 종료 방식

| 종료 방식 | 결과 |
|---|---|
| 공격으로 파괴 | 골드 + XP 지급 |
| 지형/모래 접촉으로 분해 | 모래 생성 |
| 플레이어 압착 | 피해 + 모래 생성 |
| Day 30 보스 파괴 | 런 클리어 |
| Day 30 보스 분해 후 생존 | 런 클리어 |

## 9. Sand / Weight

모래는 블록이 분해될 때 생성되는 주요 위험 자원이다.

역할:

- 전장 공간 점유
- 플레이어 이동 방해
- 무기 공격으로 제거 가능한 환경 압박 제공
- 중량 실패 조건 누적

기준:

| 항목 | 값 |
|---|---:|
| 내부 실패 기준 | `2400 cells` |
| 표시 환산 | `1 cell = 0.1 KG` |
| HUD 최대 표시 | `240.0 KG` |

이동 규칙:

- 아래로 낙하 우선
- 대각선 확산
- 플레이어 이동 중 밀림 해결
- 점프 공간 확보 시도

체력과 피해:

```text
sand_cell_max_hp = source_block_final_hp / generated_sand_cell_count
sand_damage = weapon_damage x 0.10
```

- 각 모래 셀은 독립 float HP를 가진다.
- 좌클릭 무기 공격의 근접 shape, 히트스캔, 투사체 sweep이 모래 피해를 적용한다.
- 드론 프로토콜은 모래 피해를 주지 않는다.
- `sand_cleaner`는 피해가 아니라 별도 특수 제거 효과다.
- 모래 셀은 투사체 충돌 대상이다. 비관통 투사체와 비관통 히트스캔은 첫 모래 셀에서 멈춘다.
- 관통형 공격은 블록과 모래가 같은 추가 관통 카운트를 소비한다.
- 폭발탄은 첫 충돌 지점에서 폭발하며, 폭발 범위 안의 모래 셀마다 `weapon_damage x 0.10`을 적용한다.
- 모래 제거는 골드와 골드 팝업을 지급하지 않는다.
- 기존 모래 제거 XP 누적 정책은 유지한다.

## 10. Day / Intermission / Shop

### 10-1. Day Active

Day active 중 동작:

- Day 타이머 감소
- 블록 스폰
- 보스 Day 처리
- 무기 공격과 드론 프로토콜 행동. 좌·우 무기는 좌클릭 입력을 공유하고 각자 독립 쿨타임을 사용한다.
- 채굴
- 모래 시뮬레이션
- HP 재생
- XP/골드 획득
- 실패 조건 체크

### 10-2. Day 종료

Day 1~29에서 시간이 끝나면 intermission으로 진입한다.

흐름:

1. Day active 종료
2. 스폰 중단
3. intermission 진입
4. 마지막 활성 블록 정리 대기
5. `1.25초` 뒤 키오스크 투하
6. 키오스크 투하 직후 유해물질 HUD `10초` 카운트다운 시작
7. 카운트다운 이후 단계별 유해물질 지속피해 시작
8. 키오스크 상호작용으로 상점 진입

### 10-3. Intermission 유해물질

| 상태 | 지속시간 | 초당 피해 | HUD |
|---|---:|---:|---|
| `countdown` | `10초` | `0` | 숫자 카운트다운 |
| `warning` | `10초` | `2` | `유해물질 경고` |
| `danger` | `10초` | `5` | `유해물질 위험` |
| `critical` | Next Day까지 | `10` | 짧은 주기로 바뀌는 글리치 문자열 |

- 유해물질 피해는 방어력을 무시하는 고정 환경 피해다.
- `warning`, `danger`, `critical` 동안 HP 재생과 즉시 회복을 막는다.
- ESC pause, 상점 UI, 트리 pause 동안 타이머와 피해를 진행하지 않는다.
- Next Day 전환 시작 시 상태, 타이머, 피해 누적값, 글리치 HUD를 초기화한다.

### 10-4. 키오스크

| 항목 | 값 |
|---|---:|
| 상호작용 키 | `E` |
| 상호작용 거리 | `2U` |
| 낙하 속도 | `780 px/s` |
| 투하 지연 | `1.25초` |

### 10-5. 상점

상점은 실제 구매 UI다.

현재 기능:

- 상점 아이템 5개 롤
- 아이템 목록 표시
- 상세 정보 표시
- 가격 표시
- 구매 가능 여부 표시
- 구매 처리
- 구매 성공 시 현재 상점 목록에서 제거
- `Close`
- `Next Day`

신규 장비 카테고리:

- `weapon`
- `drone`
- `drone_protocol`
- `passive_module`

상점 런타임은 `attack_module`, `function_module`, `enhance_module` 원본 데이터를 각각 `weapon`, `drone_protocol`, `passive_module`로 정규화한다.
과도기 `part`, `artifact` 분기는 legacy 호환을 위해 남아 있지만 신규 상점 UI의 기본 장비 경로는 아니다.

### 10-6. Next Day

`Next Day` 버튼을 누르면:

1. 상점 닫기
2. 키오스크 제거
3. 페이드 아웃
4. 이자 지급
5. 예약된 벽 복구가 있으면 실행
6. intermission 유해물질 상태와 HUD 초기화
7. intermission 종료
8. Day 증가
9. 새 Day 시간 설정
10. 스폰 타이머 재시작
11. 페이드 인

기본 규칙:

- 벽은 자동 복구하지 않는다.
- 모래도 기본 유지한다.
- 벽 복구는 특정 효과가 예약했을 때만 실행한다.

## 11. Shop Item

### 11-1. 카테고리

Phase 5 상점 런타임은 아래 카테고리를 사용한다.

| 카테고리 | 의미 |
|---|---|
| `weapon` | 좌측/우측 슬롯 무기 |
| `drone` | 현재는 기본 드론만 사용 |
| `drone_protocol` | 기본 드론이 실행하는 자동 행동 |
| `passive_module` | 무기/프로토콜 조건부 강화 |

### 11-2. 랭크

아이템 랭크:

- D
- C
- B
- A
- S

Day가 높아질수록 높은 랭크 비중이 증가한다.
행운은 랭크 weight를 보정한다.

랭크별 기본 가격 (`SHOP_ITEM_RANK_FALLBACK_PRICES`):

| 랭크 | 기본 가격 |
|---|---:|
| D | 15G |
| C | 30G |
| B | 60G |
| A | 120G |
| S | 240G |

아이템 데이터에 `price_gold`가 없거나 0이면 랭크 기본 가격을 사용한다.
최종 가격 결정은 `GameState.get_effective_shop_item_price()`가 담당한다.

### 11-3. 구매 결과

- 무기: 두 무기 슬롯 중 하나에 장착 또는 교체
- 드론 프로토콜: 5슬롯에 장착. 동일 프로토콜 중복 허용
- 패시브 모듈: 5슬롯에 장착. 일부 효과는 중복 적용 금지

장착 UI 규칙:

- 무기는 빈 슬롯을 먼저 채운다.
- 무기 슬롯이 가득 차면 상점 상단에서 좌측 또는 우측 슬롯을 선택한 뒤 교체한다.
- 드론 프로토콜은 빈 슬롯을 먼저 채운다.
- 드론 프로토콜 슬롯이 가득 차면 상점 상단에서 교체할 프로토콜 슬롯을 선택한다.
- 패시브 모듈은 5슬롯을 넘겨 장착하지 않는다.
- 패시브 모듈 슬롯이 가득 차면 상점 상단에서 교체할 패시브 슬롯을 선택한다.
- 패시브 교체 시 기존 source-instance 효과를 제거하고 새 패시브 효과를 적용한다.

## 14. Run End

- HP 0
