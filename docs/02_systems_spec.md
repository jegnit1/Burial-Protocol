# Burial Protocol - Systems Specification

기준일: `2026-06-05`
기준 브랜치: `main`

---

## 0. 목적

이 문서는 Burial Protocol의 실제 구현 기준 시스템 스펙을 하나로 통합한 문서다.
기존 문서와 코드에 남아 있는 `attack_module/function_module/enhance_module` 표현은 legacy 구현명으로 취급한다.

신규 시스템 설명은 아래 4분류를 기준으로 한다.

```text
Weapon   = 좌/우 슬롯 기본공격
Protocol = 드론 자동공격
Module   = 패시브 스킬
Item     = 누적 스탯 제공품
```

이 문서는 Codex 또는 개발자가 기능을 수정할 때 우선 참고하는 메인 시스템 스펙이다.

## 3. Main Scene 책임

`Main.gd`는 전투 루프와 Day 전환을 총괄한다.

주요 책임:

- 무기 공격 트리거 처리
- 프로토콜 자동공격 처리

- 채굴 처리
- 블록 파괴/분해 처리
- 모래 시뮬레이션 호출

## 4. 입력 체계

현재 기본 입력:

| 행동 | 입력 |
|---|---|
| 좌 이동 | `A`, `Left Arrow` |
| 우 이동 | `D`, `Right Arrow` |
| 점프 | `W`, `Up Arrow`, `Space` |
| 아래/급강하 | `S`, `Down Arrow` |
| 공격 | `Left Mouse Button` |
| 채굴 | `Right Mouse Button` |
| 대시 | `Z`, 좌/우/하 더블탭 |
| 상호작용 | `E` |
| 디버그 패널 | `Tab` |
| 일시정지 | `ESC` |
| 수동 종료/리스타트 | `R` |

입력 보장은 `GameConstants.ensure_input_actions()`가 담당한다.

향후 무기 좌/우 슬롯을 분리할 경우 입력은 아래 중 하나로 확장한다.

| 방식 | 설명 |
|---|---|
| 단일 공격 입력 | 현재 공격 입력이 선택/우선 무기를 발동 |
| 좌/우 분리 입력 | 좌 슬롯 무기와 우 슬롯 무기를 별도 입력으로 발동 |

확정 전까지 문서에서는 “무기 기본공격 입력”으로 표현한다.

## 6. Mining

채굴은 우클릭 기반 환경 상호작용이다.
전투 장비와 별개다.

대상:

- 모래 셀
- 좌우 벽 서브셀

비대상:

- 낙하 블록

수치:

| 항목 | 값 |
|---|---:|
| 기본 채굴 데미지 | `1` |
| 채굴 쿨다운 | `0.15초` |
| 채굴 버퍼 | `0.12초` |
| 채굴 거리 | `0.25U` |
| 채굴 높이 | `1U` |

intermission 진입 후 `3초` 유예가 지나면 채굴만 정지된다.
플레이어 이동, 모래 자연 반응, 모래 밀림은 유지한다.

### 6-1. 채굴 확장: 벽 내부 특수 객체

채굴 시스템은 향후 좌우 벽 내부의 특수 객체를 포함하도록 확장한다.

벽 내부 특수 객체는 크게 아래 두 계열로 나눈다.

| 계열 | 의미 | 처리 방식 |
|---|---|---|
| Treasure Chest | 보상형 특수 객체 | 채굴로 노출 후 `E` 상호작용 |
| Creep | 리스크형 특수 객체 | 노출 후 공격 기능으로 파괴 |

의도:

- 채굴을 단순 벽 제거가 아니라 탐사/보상/리스크 관리 루프로 확장한다.
- 중앙 전투와 좌우 벽 채굴의 역할을 분리한다.
- 채굴 빌드가 보상 탐색과 위험 대응에 의미를 가지도록 한다.

### 6-2. 보물상자 벽블록

런 중 좌우 벽 내부에는 간헐적으로 빛나는 이펙트가 표시되는 벽블록이 존재할 수 있다.

규칙:

- 빛나는 벽블록은 보물상자가 숨겨진 후보임을 의미한다.
- 해당 벽블록을 채굴하면 보물상자가 등장한다.
- 보물상자는 즉시 획득되지 않고, 월드 오브젝트로 노출된다.
- 플레이어가 보물상자 근처에서 `E` 상호작용을 하면 획득 팝업이 열린다.
- 보상 팝업이 열린 동안 게임은 일시정지된다.
- 보상 팝업에는 랜덤 보상 연출을 적용한다.

보물상자 자체 등급:

- Normal
- Silver
- Gold
- Platinum

보물상자 등급별로 아래 요소가 달라진다.

- 보물상자 이미지
- 빛나는 벽블록 이펙트 색상/강도
- 보상 테이블
- 고등급 보상 등장 확률
- 보상 저점
- 등장 위치 경향성

### 6-3. 보물상자 보상 체계

보물상자의 모든 보상은 `Weapon / Protocol / Module / Item` 체계 중 하나로 분류한다.
보상 자체는 `D~S` 등급을 가진 보상 항목으로 정의한다.

예시:

- `S급 골드주머니` = Item 계열 보상
- `B급 경험치 물약` = Item 계열 보상
- `C급 모래제거 프로토콜` = Protocol 또는 특수 보상
- `A급 무기 보급권` = Weapon 계열 보상
- `D급 패시브 모듈 부품` = Module 계열 보상

보상 대분류:

| 분류 | 예시 | 설명 |
|---|---|---|
| Gold | `D~S급 골드주머니` | 골드 지급 보상 |
| XP | `D~S급 경험치 물약` | 경험치 지급 보상 |
| Weapon | `D~S급 무기 보급권` | 좌/우 무기 슬롯 장비 후보 |
| Protocol | `D~S급 프로토콜 보급권` | 드론 자동공격 장비 후보 |
| Module | `D~S급 모듈 부품` | 패시브 모듈 후보 |
| Item | `D~S급 강화 아이템` | 누적 스탯 제공품 |
| Sand Removal | `D~S급 모래제거 보상` | 일정량의 모래를 제거하는 보상 |

모래삭제 보상은 레벨업 카드가 아니라 보물상자 보상, 상점 아이템, 특수 이벤트에서만 사용한다.

### 6-4. 크립

벽 내부에는 보물상자뿐 아니라 리스크형 특수 객체도 존재한다.
이를 개발 스펙상 `Creep`으로 정의한다.

크립 규칙:

- 크립은 좌우 벽 내부에 숨겨진 위협 오브젝트다.
- 크립은 채굴로 노출될 수 있다.
- 크립은 채굴로 파괴하지 않고 공격 기능으로 파괴한다.
- 크립을 파괴하지 않으면 지속적으로 플레이어를 방해한다.
- 크립은 노출 즉시 과도한 피해를 주지 않고, 짧은 경고/활성화 시간을 가진다.

## 7. Equipment System

### 7-1. 전체 구조

장비/성장 객체는 아래 네 계층으로 구분한다.

| 계층 | 슬롯/소지 | 발동 | 역할 |
|---|---|---|---|
| Weapon | 좌/우 슬롯 | 입력 기반 | 기본공격 |
| Protocol | 드론 최대 5개 | 자동 | 자동공격 |
| Module | 최대 5개 | 패시브 | 빌드 변형 |
| Item | 기본 제한 없음 | 패시브/누적 | 스탯 제공 |

현재 코드에 남아 있는 `attack_module`은 이 네 계층 중 하나로 재분류해야 하는 legacy 구현명이다.

### 7-2. Weapon

무기는 좌/우 슬롯에 장착하는 기본공격 장비다.

규칙:

| 항목 | 규칙 |
|---|---|
| 슬롯 | 좌 슬롯, 우 슬롯 |
| 역할 | 기본공격 |
| 발동 | 공격 입력 기반 |
| 장착 수 | 최대 2개 |
| 중복 장착 | 추후 판단. 기본적으로 같은 무기 좌/우 중복 가능 여부는 데이터가 결정 |
| 등급 | `D`, `C`, `B`, `A`, `S` |

무기 데이터는 아래 축을 가진다.

| 축 | 의미 |
|---|---|
| `attribute` | 피해/연출 속성. 예: physical, fire, electric |
| `attack_type` | 유저가 보는 공격 유형/시너지 태그. 예: projectile, area, linear, chain, explosion |
| `activation_mode` | 입력 유지/채널링 등 발동 방식 |
| `hit_model` | 실제 피해 판정 생성 방식 |
| `hit_shape` | 충돌/피해 판정용 추상 shape |
| `effect_style` | VFX/연출 선택 |
| `base_cooldown` | 공격 인스턴스 또는 피해 tick 기본 주기 |

무기 공격 대상:

- 활성 낙하 블록
- 보스 블록
- 크립 등 공격으로 처리해야 하는 위험 객체

무기 비대상:

- 일반 모래
- 일반 벽
- 플레이어
- 환경 장식물

### 7-3. Protocol

프로토콜은 드론에 장착하는 자동공격 장비다.

규칙:

| 항목 | 규칙 |
|---|---|
| 장착 위치 | 드론 |
| 최대 장착 수 | 5개 |
| 역할 | 자동공격 |
| 발동 | 공격 입력과 무관한 자동 발동 |
| 등급 | `D`, `C`, `B`, `A`, `S` |

프로토콜은 자동으로 블록을 공격하지만, 플레이어 기본공격의 역할을 빼앗으면 안 된다.
따라서 프로토콜은 아래 방향으로 밸런싱한다.

- 안정적인 보조 처리력
- 특정 조건 대응
- 다중 타겟 보조
- 빌드 완성 시 시너지 강화

프로토콜은 기본적으로 모래를 직접 제거하지 않는다.
모래 제거가 필요한 경우 특수 보상/아이템/이벤트로 제한한다.

### 7-4. Module

모듈은 최대 5개 장착하는 패시브 스킬 장비다.

규칙:

| 항목 | 규칙 |
|---|---|
| 최대 장착 수 | 5개 |
| 역할 | 패시브 스킬, 조건부 효과, 빌드 변형 |
| 직접 공격 생성 | 기본 금지 |
| 등급 | `D`, `C`, `B`, `A`, `S` |

모듈 효과 예시:

- 좌/우 무기의 속성이 다르면 데미지 증가
- 프로토콜 쿨다운 감소
- 같은 `attack_type` 무기를 장착하면 치명타 증가
- 체력이 낮을 때 방어력 증가
- 채굴 중 이동속도 감소 완화

모듈은 공격의 “주체”가 아니라 공격/생존/경제 규칙을 바꾸는 장비다.

### 7-5. Item

아이템은 기본적으로 소지 제한 없이 누적되는 스탯 제공품이다.

규칙:

| 항목 | 규칙 |
|---|---|
| 소지 제한 | 기본 없음 |
| 장착 슬롯 | 없음 |
| 역할 | 스탯 제공, 누적 성장 |
| 제한 예외 | 특정 아이템은 unique 또는 max_stack 가능 |
| 등급 | `D`, `C`, `B`, `A`, `S` |

아이템 효과 예시:

- 데미지 +n%
- 공격속도 +n%
- 최대 체력 +n
- 행운 +n
- 이자율 +n%p
- 배터리 회복 +n/sec

단순 스탯 증가는 모듈보다 아이템에 우선 배치한다.

### 7-6. legacy attack_module 처리

현재 코드/데이터의 `attack_module`은 다음 기준으로 분리한다.

| 기존 성격 | 신규 분류 |
|---|---|
| 좌클릭 기반 melee/ranged 공격 | Weapon |
| 자동 타겟팅/자동 공격 mechanic | Protocol |
| 공격을 직접 만들지 않는 보조 효과 | Module |
| 순수 스탯 증가 | Item |

기존 공격모듈 합성 규칙은 신규 체계에 그대로 복사하지 않는다.
무기/프로토콜/모듈/아이템 각각의 성장 규칙은 별도로 판단한다.

### 7-7. 치명타

- 기본 치명타 확률: `1%`
- 치명타 배율: `200%`

치명타는 공격 판정에 적용된다.
채굴에는 적용되지 않는다.

## 8. Block / Spawn

### 8-1. 최신 블록 모델

최신 블록 모델:

```text
Runtime Block = Material x Size + optional Type
```

- Material = 재질/성질
- Size = 가로/세로 크기
- Type = 선택 affix/modifier

기존 코드에 `base` 명칭이 남아 있을 수 있으나 문서상 기준은 material/size/type이다.

### 8-2. 스폰 흐름

현재 라이브 스폰은 v1 흐름이다.

1. 현재 난이도와 Day 확인
2. `BlockCatalog`에서 유효 candidate 수집
3. candidate weight 기반 랜덤 선택
4. optional type 선택
5. `BlockSpawnResolver`가 resolved definition 생성
6. `BlockData`로 변환
7. `FallingBlock` 생성

### 8-3. 최종 HP

```text
final_hp =
  BLOCK_HP_PER_UNIT
  x size_hp_multiplier
  x material_hp_multiplier
  x difficulty_hp_multiplier
  x type_hp_multiplier
  x day_hp_multiplier
```

기본값:

- `BLOCK_HP_PER_UNIT = 10.0`

### 8-4. 최종 보상

```text
final_reward =
  BLOCK_REWARD_PER_UNIT
  x size_reward_multiplier
  x material_reward_multiplier
  x type_reward_multiplier
```

기본값:

- `BLOCK_REWARD_PER_UNIT = 5.0`

### 8-5. 최종 모래량

```text
final_sand_units =
  BLOCK_SAND_UNITS_PER_UNIT
  x size_reward_multiplier
  x type_sand_units_multiplier
```

기본값:

- `BLOCK_SAND_UNITS_PER_UNIT = 36.0`

### 8-6. 블록 종료 방식

| 종료 방식 | 결과 |
|---|---|
| 공격으로 파괴 | 골드 + XP 지급 |
| 지형/모래 접촉으로 분해 | 모래 생성 |
| 플레이어 압착 | 피해 + 모래 생성 |
| Day 30 보스 파괴 | 런 클리어 |
| Day 30 보스 분해 후 생존 | 런 클리어 |

## 10. Day / Intermission / Shop

### 10-1. Day Active

Day active 중 동작:

- Day 타이머 감소
- 블록 스폰
- 보스 Day 처리
- 무기 공격
- 프로토콜 자동공격
- 모듈/아이템 효과 평가
- 채굴
- 모래 시뮬레이션
- HP 재생
- XP/골드 획득
- 실패 조건 체크

### 10-2. Day 종료

Day 1~29에서 시간이 끝나면 intermission으로 진입한다.

흐름:

1. Day active 종료
2. 스폰 중단
3. intermission 진입
4. 마지막 활성 블록 정리 대기
5. `1.25초` 뒤 키오스크 투하
6. `3초` 뒤 채굴 잠금
7. 키오스크 상호작용으로 상점 진입

### 10-3. 키오스크

| 항목 | 값 |
|---|---:|
| 상호작용 키 | `E` |
| 상호작용 거리 | `2U` |
| 낙하 속도 | `780 px/s` |
| 투하 지연 | `1.25초` |

### 10-4. 상점

상점은 실제 구매 UI다.

현재 기능:

- 상점 아이템 5개 롤
- 아이템 목록 표시
- 상세 정보 표시
- 가격 표시
- 구매 가능 여부 표시
- 구매 처리
- 구매 성공 시 현재 상점 목록에서 제거
- 골드 지불 reroll
- 슬롯별 lock
- lock된 슬롯은 추가 보상이 아니라 5개 상점 슬롯 중 해당 슬롯을 보존
- `Close`
- `Next Day`

상점 신규 카테고리:

| 카테고리 | 의미 |
|---|---|
| `weapon` | 좌/우 슬롯 기본공격 장비 |
| `protocol` | 드론 자동공격 장비 |
| `module` | 패시브 스킬 장비 |
| `item` | 소지 제한 없는 스탯 제공품 |

legacy 카테고리 매핑:

| legacy | migration 기준 |
|---|---|
| `attack_module` | `weapon` 또는 `protocol` |
| `function_module` | `module` 또는 특수 `item` |
| `enhance_module` | 대부분 `item`, 일부 `module` |

### 10-5. Next Day

`Next Day` 버튼을 누르면:

1. 상점 닫기
2. 키오스크 제거
3. 페이드 아웃
4. 이자 지급
5. 예약된 벽 복구가 있으면 실행
6. intermission 종료
7. Day 증가
8. 새 Day 시간 설정
9. 스폰 타이머 재시작
10. 페이드 인

기본 규칙:

- 벽은 자동 복구하지 않는다.
- 모래도 기본 유지한다.
- 벽 복구는 특정 효과가 예약했을 때만 실행한다.

## 11. Shop Item / Reward

### 11-1. 신규 카테고리

| 카테고리 | 의미 | 구매 결과 |
|---|---|---|
| `weapon` | 기본공격 장비 | 좌/우 무기 슬롯에 장착 또는 교체 후보 |
| `protocol` | 자동공격 장비 | 드론 프로토콜 슬롯에 장착 |
| `module` | 패시브 장비 | 모듈 슬롯에 장착 |
| `item` | 스탯 제공품 | 즉시 소지/스택 반영 |

### 11-2. 랭크

상점 상품 랭크:

- D
- C
- B
- A
- S

Day가 높아질수록 높은 랭크 비중이 증가한다.
행운은 랭크 weight를 보정한다.

랭크별 기본 가격 (`SHOP_ITEM_RANK_FALLBACK_PRICES`):

| 랭크 | 기본 가격 |
|---|---:|
| D | 15G |
| C | 30G |
| B | 60G |
| A | 120G |
| S | 240G |

아이템 데이터에 `price_gold`가 없거나 0이면 랭크 기본 가격을 사용한다.
최종 가격 결정은 `GameState.get_effective_shop_item_price()`가 담당한다.

### 11-3. 구매 결과 원칙

- 무기: 좌/우 슬롯에 장착하거나 교체 후보로 제시한다.
- 프로토콜: 드론 프로토콜 슬롯에 장착한다. 최대 5개 제한을 따른다.
- 모듈: 모듈 슬롯에 장착한다. 최대 5개 제한을 따른다.
- 아이템: 소지 목록 또는 스택에 즉시 반영한다. 기본 소지 제한은 없다.

## 14. Run End

실패 조건:

- HP 0
- 모래 중량 한도 초과

- 수동 종료

런 종료 시:

1. `GameState.finish_temporary_run()`
2. 결과 데이터 저장
