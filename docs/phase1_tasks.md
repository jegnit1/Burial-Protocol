# Burial Protocol - Phase 1 Tasks

## 0. 문서 목적

이 문서는 현재 Phase 1 기준으로 무엇이 이미 구현되었고,
무엇이 아직 제한적 또는 TODO 상태인지 정리한다.

기준일: `2026-04-20`

현재 Phase 1은 더 이상 단순 체크리스트 단계가 아니라,
"플레이 가능한 코어 루프를 유지한 채 누락 시스템을 메우는 단계"로 본다.

---

## 1. 현재 완료된 항목

### 1-1. 상위 메뉴 흐름

- `Title -> MainHub -> Main -> Result -> MainHub` 흐름
- 캐릭터 선택 화면 연결
- 난이도 선택과 해금 구조
- 결과 화면 기록 출력
- 업적/성장/아이템 목록 placeholder 화면

### 1-2. 저장 및 허브 상태

- `user://profile.save` 기반 저장
- 선택 캐릭터 저장
- 마지막 선택 난이도 저장
- 난이도 클리어 기록 저장
- 캐릭터별 최고 기록 저장

### 1-3. 코어 월드/전투 루프

- 고정 월드 레이아웃
- 중앙 전장 + 좌우 채굴 벽 구조
- 낙하 블록 스폰
- Rush Day / Boss Day
- Day 타이머
- Day 30 클리어/실패 분기

### 1-4. 플레이어 액션

- 이동
- 점프 / 코요테 타임 / 점프 버퍼
- 추가 점프
- 대시
- 벽타기
- 공격
- 채굴
- 낙하 블록에 의한 압착 피해

### 1-5. 모래/벽 시스템

- active-cell 기반 모래 시뮬레이션
- 블록 분해 시 모래 생성
- 좌우 벽 서브셀 채굴
- 플레이어와 모래의 밀림 상호작용
- 중량 초과 실패

### 1-6. 성장과 런타임 스탯

- 블록 파괴/모래 제거 XP 획득
- 레벨업 카드 3택
- 공격력/공속/최대체력/이속/채굴데미지/채굴속도 카드
- 공격범위/치명타확률/방어력/HP재생/점프력/채굴범위 카드
- 치명타
- HP 재생
- 이자 지급
- ESC 스탯 확인 패널

### 1-7. Day 전환 루프

- Day 종료 -> intermission 진입
- 마지막 활성 블록 정리 대기
- 지연 후 키오스크 투하
- 키오스크 안착 후 상호작용
- 상점 UI 진입
- `Next Day` 버튼 연결
- 페이드 인/아웃

### 1-8. HUD / 런 UI

- 상단 Day/시간/난이도 표시
- 골드/중량 HUD
- HP/XP 표시
- 배터리 표시
- 우측 하단 스킬 슬롯 3칸
- 첫 번째 슬롯 대시 쿨다운 연동
- ESC 일시정지 + 스탯 리스트

### 1-9. 데이터 구조 정리

- 블록 콘텐츠의 `.tres` 분리
- Stage/Day 콘텐츠의 `.tres` 분리
- `GameData.gd` 로딩 진입점 추가
- Block Base / Block Type 개념 정리

## 2. 현재 제한적 구현 또는 TODO

### 2-1. 상점

현재 상점은 "진입과 Next Day 전환"만 연결되어 있다.

- 실제 구매 품목 없음
- 경제 선택지 없음
- 벽 초기화 아이템 미구현

### 2-2. 메타 시스템

아래는 구조만 있거나 placeholder 상태다.

- 업적 실기능
- 성장 실기능
- 아이템 목록 실기능
- 영구 재화 활용

### 2-3. 블록 특수 결과

데이터 훅은 존재하지만 실제 게임플레이 영향은 제한적이다.

예:

- bonus gold
- glass shatter damage
- explosion

### 2-4. 행운

- 스탯 표시에는 포함
- 현재 효과 없음
- 추후 아이템/상자 시스템용 TODO

## 3. 현재 우선순위

### 우선순위 A - 상점 구매 실기능

다음 단계에서 가장 자연스럽게 이어질 작업:

- 상점 품목 정의
- 구매 효과 적용
- 벽 초기화 예외 구매
- 경제 밸런스 연결

### 우선순위 B - 블록 특수 결과 구체화

- Base/Type 특수 결과의 실제 적용
- 시각 피드백 확장
- 폭탄/보너스 골드 효과 연결

### 우선순위 C - 메타 시스템 placeholder 해소

- 업적 실제 조건
- 성장 실제 효과
- 인벤토리/아이템 보유 구조

---

## 4. 이번 시점에서 지켜야 할 것

다음 작업에서도 유지해야 하는 핵심:

- 이동 감각
- 공격/채굴 리듬
- 모래 물리 반응
- Day 종료 -> 상점 -> Next Day 루프
- HUD/ESC의 가시성
- `.tres` 기반 콘텐츠 소유권

즉, 현재 Phase 1의 목표는
"이미 돌아가는 루프를 갈아엎지 않고, 비어 있는 연결부를 하나씩 메우는 것"이다.

---

## 5. 체크리스트

다음 작업 전/후 최소 확인 항목:

- Day 종료가 결과 화면이 아니라 intermission으로 가는가
- 키오스크가 실제로 안착 후 상호작용 가능한가
- 상점 UI에서 `Next Day`가 동작하는가
- 기본 Stage 전환에서 벽이 유지되는가
- ESC 스탯 패널이 현재 최종값을 보여주는가
- 레벨업 카드 풀이 실제 성장 가능한 스탯과 맞는가
- 문서가 현재 코드와 동기화되어 있는가

---

## 6. Phase 1 요약

현재 Burial Protocol의 Phase 1은 아래처럼 요약할 수 있다.

"전투/채굴/모래/성장/Day 전환의 코어 루프는 이미 플레이 가능하며,
상점 구매와 메타 시스템이 아직 제한적으로 남아 있는 상태."
