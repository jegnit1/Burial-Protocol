# Burial Protocol - Game Design Document

기준일: `2026-04-28`  
기준 브랜치: `main`  
엔진: Godot `4.6`  
언어: GDScript

---

## 1. 게임 한 줄 정의

**Burial Protocol**은 좁은 수직 샤프트 안에서 위에서 떨어지는 블록을 파괴하거나 피하고, 쌓이는 모래와 중량 압박을 버티며, Day마다 상점에서 장비와 능력을 강화해 Day 30까지 생존하는 2D 사이드뷰 생존 로그라이트 게임이다.

---

## 2. 핵심 플레이 감성

Burial Protocol의 핵심 감성은 아래 세 가지다.

1. 위에서 떨어지는 블록을 빠르게 판단하고 처리한다.
2. 처리하지 못한 블록은 모래로 분해되어 전장을 점점 막는다.
3. Day 사이 상점에서 공격모듈과 아이템을 구매해 다음 Day를 버틴다.

즉, 이 게임은 단순 낙하물 회피 게임이 아니라 **낙하 블록 처리 + 모래 압박 + Day별 장비 성장**이 결합된 세로형 생존 루프다.

---

## 3. 현재 구현 상태 요약

현재 프로젝트는 플레이 가능한 코어 루프를 가진 상태다.

구현 완료 또는 실제 연결된 영역:

- Title -> MainHub -> Main -> Result 흐름
- 캐릭터 선택 placeholder와 기본 일꾼 선택
- 난이도 선택과 순차 해금 구조
- 30 Day 런 구조
- 낙하 블록 스폰
- 보스 Day 스폰
- 블록 파괴/분해
- 모래 생성과 모래 시뮬레이션
- 중량 한도 기반 실패
- 좌우 벽 채굴
- 마우스 방향 기반 채굴
- 점프, 추가 점프, 대시, 벽타기
- 공격모듈 기반 공격
- melee/ranged/mechanic 공격모듈 타입
- Day 종료 후 intermission
- 낙하형 키오스크
- Day 상점 UI
- 상점 아이템 5개 롤
- attack_module/function_module/enhance_module 구매
- 공격모듈 즉시 장착, 중복 장착, 합성
- XP와 레벨업 카드
- 레벨업 카드 5장 제시 방식
- 근거리/원거리 공격력 분리 (`run_bonus_melee_attack_damage`, `run_bonus_ranged_attack_damage`)
- 공격모듈 attack_style/effect_style 시스템 (AttackModuleStyleResolver)
- 상점 아이템 랭크별 가격 티어링 (`SHOP_ITEM_RANK_FALLBACK_PRICES`)
- 런타임 스탯 증가
- HUD와 ESC 스탯 패널
- 저장 파일과 최고 기록 저장

아직 완성 시스템으로 보지 않는 영역:

- 설정 메뉴 실기능
- 메타 성장 실효과
- 업적 실효과
- 인벤토리/도감성 UI
- 블록 특수 결과의 모든 효과
- 최종 상점 UX/비주얼/한글화
- 최종 아트 적용
- 최종 밸런스

## 6. 플레이어

### 6-1. 역할

플레이어는 좁은 중앙 샤프트 안에서 블록을 공격하고, 벽과 모래를 채굴하며, 쌓이는 모래를 피해 생존하는 주체다.

전투 능력의 핵심은 플레이어 자체가 아니라 **장착된 공격모듈**이다.
플레이어는 공격모듈을 운용하는 플랫폼에 가깝다.

### 6-2. 이동

현재 이동 시스템:

- 좌우 이동
- 공중 이동
- 모래 속 감속
- 점프
- 추가 점프
- 코요테 타임
- 점프 버퍼
- 빠른 낙하
- 좌/우/하 대시
- 배터리 기반 벽타기

주요 수치:

| 항목 | 값 |
|---|---:|
| 지상 이동속도 | `426 px/s` |
| 공중 이동속도 | `373 px/s` |
| 모래 속 이동 배율 | `0.62` |
| 중력 | `2000 px/s^2` |
| 점프 속도 | `-853` |
| 추가 점프 | `1회` |
| 벽 점프 가로 속도 | `480` |
| 빠른 낙하 가속 | `2933 px/s^2` |
| 빠른 낙하 최대 속도 | `1306 px/s` |
| 대시 거리 | `4U` |
| 대시 지속시간 | `0.08초` |
| 대시 쿨다운 | `0.45초` |
| 벽타기 배터리 | `100` |
| 벽타기 소모 | `5/sec` |
| 기본 배터리 회복 | `5/sec` |

### 6-3. 벽타기

벽타기는 고정벽 전용 이동 시스템이다.

발동 조건:

- 대시 중이 아님
- 배터리 `> 0`
- 고정벽 접촉
- 접촉한 벽 방향 입력 유지
- 플레이 가능 상단 제한 아래에 있음

벽타기 중에는 낙하 속도가 제한되고 배터리가 소모된다.
벽타기가 아닐 때는 배터리가 회복된다.

## 7. 공격모듈 시스템

### 7-1. 정의

공격모듈은 캐릭터가 장착하는 무기 아이템이다.
공격의 주체는 캐릭터의 기본 공격 하나가 아니라 장착된 모듈들이다.

현재 공격모듈은 시각적으로 플레이어 주변을 공전하며, 실제 판정 기준은 캐릭터 위치를 사용한다.

### 7-2. 장착 규칙

| 항목 | 규칙 |
|---|---|
| 최대 장착 수 | `5개` |
| 중복 장착 | 가능 |
| 구매 결과 | 즉시 장착 |
| 슬롯이 남아 있음 | 새 모듈 추가 장착 |
| 슬롯이 가득 참 | 기본 구매 불가 |
| 예외 | 동일 모듈/동일 등급 합성 가능 시 구매 허용 |
| 등급 | `D`, `C`, `B`, `A`, `S` |

### 7-3. 합성 규칙

합성 조건:

```text
동일 모듈 + 동일 등급 2개
```

합성 결과:

```text
상위 등급 동일 모듈 1개
```

예:

```text
D 단검 + D 단검 = C 단검
C 소드 + C 소드 = B 소드
```

### 7-4. 타입

| 타입 | 발동 방식 | 현재 구현 성격 |
|---|---|---|
| `melee` | 좌클릭 기반 | 캐릭터 기준 근접 shape query |
| `ranged` | 좌클릭 기반 | 투사체 또는 레이저 히트스캔 |
| `mechanic` | 좌클릭 무관 | 자동 타겟팅/자동 공격 |

### 7-5. 공격 대상

공격모듈의 공격 대상은 블록이다.

대상:

- 낙하 중인 블록
- 활성 블록
- 보스 블록

비대상:

- 모래
- 벽
- 플레이어 자신
- 환경 장식물

## 8. 채굴 시스템

채굴은 우클릭 기반 환경 상호작용이다.
공격모듈과 별개의 시스템이다.

현재 채굴 대상:

- 모래 셀
- 좌우 벽 서브셀

현재 채굴 비대상:

- 낙하 블록

주요 수치:

| 항목 | 값 |
|---|---:|
| 기본 채굴 데미지 | `1` |
| 채굴 쿨다운 | `0.15초` |
| 채굴 버퍼 | `0.12초` |
| 채굴 거리 | `0.25U` |
| 채굴 높이 | `1U` |

Day 종료 후 intermission 유예 시간이 지나면 채굴만 정지된다.
플레이어 이동, 자연 모래 반응, 플레이어의 모래 밀림은 유지된다.

## 9. 블록 시스템

### 9-1. 최신 블록 모델

최신 기준 블록은 아래 구조다.

```text
Runtime Block = Material x Size + optional Type
```

- `Material`: 재질/성질
- `Size`: 가로/세로 크기
- `Type`: 선택 affix/modifier

기존 코드 호환용으로 `base`라는 이름이 일부 남아 있을 수 있지만, 문서상 개념은 material/size/type을 기준으로 한다.

### 9-2. 스폰 흐름

1. 현재 난이도와 Day 확인
2. `BlockCatalog`에서 유효한 spawn candidate 수집
3. candidate weight 기반 랜덤 선택
4. optional type 선택
5. `BlockSpawnResolver`가 resolved definition 생성
6. `BlockData`로 변환
7. `FallingBlock` 생성

### 9-3. 최종 HP 공식

```text
final_hp =
  BLOCK_HP_PER_UNIT
  x size_hp_multiplier
  x material_hp_multiplier
  x difficulty_hp_multiplier
  x type_hp_multiplier
```

기본값:

- `BLOCK_HP_PER_UNIT = 10.0`

### 9-4. 최종 보상 공식

```text
final_reward =
  BLOCK_REWARD_PER_UNIT
  x size_reward_multiplier
  x material_reward_multiplier
  x type_reward_multiplier
```

기본값:

- `BLOCK_REWARD_PER_UNIT = 5.0`

### 9-5. 최종 모래량 공식

```text
final_sand_units =
  BLOCK_SAND_UNITS_PER_UNIT
  x size_reward_multiplier
  x type_sand_units_multiplier
```

기본값:

- `BLOCK_SAND_UNITS_PER_UNIT = 36.0`

### 9-6. 블록 종료 방식

| 종료 방식 | 결과 |
|---|---|
| 공격으로 파괴 | 골드 + XP 지급 |
| 지형/모래에 닿아 분해 | 모래 생성 |
| 플레이어 압착 | 플레이어 피해 + 모래 생성 |
| Day 30 보스 파괴 | 런 클리어 |
| Day 30 보스 분해 후 생존 | 런 클리어 |

중량 기준:

| 항목 | 값 |
|---|---:|
| 내부 실패 기준 | `2400 cells` |
| 표시 환산 | `1 cell = 0.1 KG` |
| HUD 최대 표시 | `240.0 KG` |

현재 모래는 자연 낙하, 대각선 확산, 플레이어 밀림, 점프 공간 확보 등을 처리한다.

## 11. Day / Intermission / Shop 루프

### 11-1. Day Active

Day active 중 동작:

- Day 타이머 감소
- 블록 스폰
- 보스 Day 처리
- 공격모듈 공격
- 채굴
- 모래 시뮬레이션
- HP 재생
- XP/골드 획득
- 실패 조건 체크

### 11-2. Day 종료

Day 1~29에서 시간이 끝나면 결과 화면으로 가지 않고 intermission으로 진입한다.

흐름:

1. Day active 종료
2. 일반 스폰 중단
3. intermission 진입
4. 마지막 활성 블록 정리 대기
5. `1.25초` 뒤 키오스크 투하
6. `3초` 뒤 채굴 잠금
7. 키오스크 상호작용으로 상점 진입

### 11-3. 키오스크

키오스크는 월드 오브젝트로 낙하한다.

| 항목 | 값 |
|---|---:|
| 상호작용 키 | `E` |
| 상호작용 거리 | `2U` |
| 낙하 속도 | `780 px/s` |
| 투하 지연 | `1.25초` |

### 11-4. 상점

상점은 intermission 중 진입 가능한 실제 구매 UI다.

현재 기능:

- 상점 아이템 5개 롤
- 아이템 목록 표시
- 상세 정보 표시
- 가격 표시
- 구매 가능 여부 표시
- 골드 차감
- 구매 성공 시 현재 상점 목록에서 제거
- `Close`
- `Next Day`

상점 카테고리:

- `attack_module`
- `function_module`
- `enhance_module`

### 11-5. Next Day

`Next Day` 버튼을 누르면:

1. 페이드 아웃
2. 이자 지급
3. 예약된 벽 복구가 있으면 실행
4. intermission 종료
5. Day 증가
6. 새 Day 시간 설정
7. 스폰 타이머 재시작
8. 페이드 인

기본 규칙:

- 벽은 자동 복구하지 않는다.
- 모래도 기본적으로 유지된다.
- 벽 복구는 특정 효과가 예약했을 때만 실행된다.

## 13. 성장 시스템

### 13-1. XP

XP 획득:

- 블록 파괴 XP
- 모래 제거 XP

### 13-2. 레벨업

레벨업 시 게임이 일시정지되고 `LevelUpUI`가 뜬다.
카드 5장 중 하나를 선택하면 즉시 런타임 스탯이 증가한다.

### 13-3. 현재 주요 스탯

- 공격력
- 공격속도
- 공격범위
- 치명타 확률
- 치명타 배율
- 최대 체력
- 방어력
- HP 재생
- 이동속도
- 점프력
- 채굴 데미지
- 채굴 속도
- 채굴 범위
- 이자율
- 행운
- 최대 중량
- 배터리 회복

## 14. 경제 시스템

현재 런 경제의 중심은 골드다.

골드 획득:

- 블록 파괴
- 일부 아이템/효과

골드 사용:

- Day 상점 구매

Next Day 전환 시 이자가 지급된다.

```text
interest = floor(current_gold x interest_rate)
```

기본 이자율:

- `STAGE_INTEREST_RATE = 0.10`

## 16. 실패 조건과 클리어 조건

### 16-1. 실패 조건

현재 실패 조건:

- HP 0
- 모래 중량 한도 초과
- Day 30 시간 종료
- 수동 종료

### 16-2. 클리어 조건

Day 30 보스를 처리하면 클리어된다.

클리어 케이스:

1. Day 30 보스를 직접 파괴
2. Day 30 보스가 모래로 분해되었지만 플레이어가 생존하고 중량 초과가 아님

## 18. 현재 개발 우선순위

우선순위 높은 작업:

1. 현재 문서와 코드 동기화 유지
2. 상점 아이템 효과 정리
3. 공격모듈 밸런스와 시각 피드백 강화
4. 블록 material/size/type 데이터 정리
5. HUD와 상점 UI 한글화/가독성 개선
6. 기본 아트 방향 확정
7. 메타 성장과 업적은 이후 단계에서 본격화

## 19. 설계 원칙 요약

- 코어 루프를 깨지 않는다.
- 상점은 현재 구현된 시스템으로 취급한다.
- 공격은 공격모듈 중심으로 설명한다.
- 블록은 material x size + optional type으로 설명한다.
- 모래는 단순 이펙트가 아니라 실패 조건과 전장 압박의 핵심이다.
- Day 전환에서 벽과 모래는 기본 유지한다.
- 메타 시스템은 placeholder 범위를 명확히 표시한다.
