# Burial Protocol - Game Design Document

기준일: `2026-06-01`
기준 브랜치: `main`
엔진: Godot `4.6`
언어: GDScript

---

## 1. 게임 한 줄 정의

**Burial Protocol**은 좁은 수직 샤프트 안에서 위에서 떨어지는 블록을 파괴하거나 피하고, 쌓이는 모래와 중량 압박을 버티며, Day마다 상점에서 장비와 능력을 강화해 Day 30까지 생존하는 2D 사이드뷰 생존 로그라이트 게임이다.

---

## 2. 핵심 플레이 감성

Burial Protocol의 핵심 감성은 아래 세 가지다.

1. 위에서 떨어지는 블록을 빠르게 판단하고 처리한다.
2. 처리하지 못한 블록은 모래로 분해되어 전장을 점점 막는다.
3. Day 사이 상점에서 무기, 드론 프로토콜, 패시브 모듈을 구매해 다음 Day를 버틴다.

즉, 이 게임은 단순 낙하물 회피 게임이 아니라 **낙하 블록 처리 + 모래 압박 + Day별 장비 성장**이 결합된 세로형 생존 루프다.

- 블록 특수 결과의 모든 효과
- 최종 상점 UX/비주얼/한글화

## 6. 플레이어

### 6-1. 역할

플레이어는 좁은 중앙 샤프트 안에서 블록을 공격하고, 좌우 벽을 채굴하며, 쌓이는 모래를 피해 생존하는 주체다.

전투 능력의 핵심은 플레이어 자체가 아니라 **장착된 무기와 드론 프로토콜**이다.
플레이어는 장비를 운용하고, 패시브 모듈 조합으로 빌드를 완성한다.

### 6-2. 이동

현재 이동 시스템:

- 좌우 이동
- 공중 이동
- 모래 속 감속
- 점프
- 추가 점프
- 코요테 타임
- 점프 버퍼
- 빠른 낙하
- 마우스 방향 360도 대시
- 대시 종료 직후 짧은 입력 유예

주요 수치:

| 항목 | 값 |
|---|---:|
| 지상 이동속도 | `426 px/s` |
| 공중 이동속도 | `373 px/s` |
| 모래 속 이동 배율 | `0.62` |
| 중력 | `2000 px/s^2` |
| 점프 속도 | `-853` |
| 추가 점프 | `1회` |
| 빠른 낙하 가속 | `2933 px/s^2` |
| 빠른 낙하 최대 속도 | `1306 px/s` |
| 대시 거리 | `GameConstants.PLAYER_DASH_DISTANCE` |
| 대시 지속시간 | `0.08초` |
| 대시 쿨다운 | `0.45초` |
| 대시 입력 유예 | `0.10초` |
| 기본 배터리 회복 | `1/sec` |

### 6-3. 벽타기

벽타기 기능은 삭제되었다.
좌우 벽 방향 입력은 이동에만 사용하며, 채굴 방향은 마우스 위치로 정한다.

## 7. 장비 시스템

### 7-1. Phase 1 확정 구조

| 장비 | 슬롯 | 규칙 |
|---|---:|---|
| 무기 | `2개` | 좌측/우측 슬롯. 좌클릭 시 장착된 무기가 각자 쿨타임에 따라 발사 |
| 드론 | `1개` | 현재는 등급 없는 기본 드론만 사용 |
| 드론 프로토콜 | `5개` | 자동 행동. 동일 프로토콜 중복 장착 가능. `D~S` 등급 |
| 패시브 모듈 | `5개` | 무기/프로토콜에 추가 효과 부여. `D~S` 등급. 일부는 중복 적용 불가 |

우측 무기 슬롯은 두 번째 무기 슬롯이지 우클릭 입력 슬롯이 아니다.
우클릭은 좌우 벽 채굴에만 사용한다. 모래는 좌클릭 무기 공격으로 제거한다.

### 7-2. 속성과 유형

장비 효과는 `속성(attribute)`과 `유형(attack_type)`을 조합해 표현한다.

| 구분 | 값 |
|---|---|
| 속성 | `electric`, `fire`, `physical`, `energy`, `chemical`, `none` |
| 유형 | `support`, `projectile`, `area`, `beam`, `chain`, `explosion` |

### 7-3. 신규 전투 스탯

| 스탯 | 역할 |
|---|---|
| 무기 공격력 | 무기 피해 flat 보너스 |
| 드론 공격력 | 드론 프로토콜 피해 flat 보너스 |
| 공격속도 | 무기 쿨타임 감소 |
| 드론 쿨타임 감소 | 프로토콜 쿨타임 감소. 희귀 스탯이며 레벨업 카드 기본 풀에서 제외 |

기존 근거리 공격력과 원거리 공격력은 제거 대상이다.

### 7-4. 현재 런타임과의 관계

현재 코드의 공격모듈 5개 장착, `melee/ranged/mechanic`, 캐릭터 주변 공전 구조는 legacy 구현이다.
다음 Phase에서 신규 장비 체계로 교체하며, 두 구조를 동시에 활성화하지 않는다.

### 7-5. 공격 대상

무기의 기본 공격 대상은 블록과 모래다.
공격형 드론 프로토콜의 기본 공격 대상은 블록이며 모래 피해를 주지 않는다.

대상:

- 낙하 중인 블록
- 활성 블록
- 보스 블록

비대상:

- 벽
- 플레이어 자신
- 환경 장식물

## 8. 채굴 시스템

채굴은 우클릭 유지와 마우스 방향으로 처리되는 환경 상호작용이다.
공격모듈과 별개의 시스템이다.

현재 채굴 대상:

- 좌우 벽 서브셀

현재 채굴 비대상:

- 낙하 블록
- 활성 블록
- 보스 블록

주요 수치:

| 항목 | 값 |
|---|---:|
| 기본 채굴 데미지 | `1` |
| 채굴 기본/최소 주기 | `0.90초 / 0.30초` |
| 채굴 배터리 소모 | `5` |
| 채굴 거리 | `0.5U` |
| 채굴 높이 | `0.5U` |

채굴 1회는 벽 채굴 데미지와 파편 처리 경로만 검사한다.
채굴 데미지, 채굴속도, 채굴범위 스탯은 좌우 벽에만 적용한다.

Day 종료 후 intermission에서도 벽 채굴과 무기 기반 모래 제거는 계속 가능하다.
플레이어 이동, 공격, 자연 모래 반응, 플레이어의 모래 밀림도 유지된다.
대신 마지막 활성 블록 정리 후 키오스크가 투하되면 중앙 상단에 유해물질 카운트다운이 표시되고, 이후 오래 머무르면 단계별 고정 환경 피해를 받는다.

## 9. 블록 시스템

### 9-1. 최신 블록 모델

최신 기준 블록은 아래 구조다.

```text
Runtime Block = Material x Size + optional Type
```

- `Material`: 재질/성질
- `Size`: 가로/세로 크기
- `Type`: 선택 affix/modifier

기존 코드 호환용으로 `base`라는 이름이 일부 남아 있을 수 있지만, 문서상 개념은 material/size/type을 기준으로 한다.

### 9-2. 스폰 흐름

1. 현재 난이도와 Day 확인
2. `BlockCatalog`에서 유효한 spawn candidate 수집
3. candidate weight 기반 랜덤 선택
4. optional type 선택
5. `BlockSpawnResolver`가 resolved definition 생성
6. `BlockData`로 변환
7. `FallingBlock` 생성

### 9-3. 최종 HP 공식

```text
final_hp =
  BLOCK_HP_PER_UNIT
  x size_hp_multiplier
  x material_hp_multiplier
  x difficulty_hp_multiplier
  x type_hp_multiplier
```

기본값:

- `BLOCK_HP_PER_UNIT = 10.0`

### 9-4. 최종 보상 공식

```text
final_reward =
  BLOCK_REWARD_PER_UNIT
  x size_reward_multiplier
  x material_reward_multiplier
  x type_reward_multiplier
```

기본값:

- `BLOCK_REWARD_PER_UNIT = 5.0`

### 9-5. 최종 모래량 공식

```text
final_sand_units =
  BLOCK_SAND_UNITS_PER_UNIT
  x size_reward_multiplier
  x type_sand_units_multiplier
```

기본값:

- `BLOCK_SAND_UNITS_PER_UNIT = 36.0`

### 9-6. 블록 종료 방식

| 종료 방식 | 결과 |
|---|---|
| 공격으로 파괴 | 골드 + XP 지급 |
| 지형/모래에 닿아 분해 | 모래 생성 |
| 플레이어 압착 | 플레이어 피해 + 모래 생성 |
| Day 30 보스 파괴 | 런 클리어 |
| Day 30 보스 분해 후 생존 | 런 클리어 |

## 10. 모래 시스템

모래는 블록이 분해될 때 생성되는 주요 위험 자원이다.

역할:

- 전장 공간 점유
- 플레이어 이동 방해
- 무기 공격으로 제거 가능한 환경 압박 제공
- 중량 실패 조건 누적

중량 기준:

| 항목 | 값 |
|---|---:|
| 내부 실패 기준 | `2400 cells` |
| 표시 환산 | `1 cell = 0.1 KG` |
| HUD 최대 표시 | `240.0 KG` |

현재 모래는 자연 낙하, 대각선 확산, 플레이어 밀림, 점프 공간 확보 등을 처리한다.

모래 셀은 원본 블록의 실제 최종 HP를 생성된 셀 수로 나눈 개별 float HP를 가진다.
좌클릭 무기 공격은 모래 셀마다 무기 피해의 `10%`를 적용한다.
모래 셀은 투사체 충돌 대상이며, 비관통 투사체와 비관통 빔은 첫 모래 셀에서 멈춘다.
관통형 공격은 블록과 모래가 같은 추가 관통 카운트를 소비한다.
드론 프로토콜은 모래 피해를 줄 수 없다. `sand_cleaner`는 피해가 아닌 별도 특수 제거 효과다.
모래 제거는 골드를 지급하거나 골드 팝업을 만들지 않는다. 기존 모래 제거 XP 정책은 유지한다.

## 11. Day / Intermission / Shop 루프

### 11-1. Day Active

Day active 중 동작:

- Day 타이머 감소
- 블록 스폰
- 보스 Day 처리
- 좌·우 무기 공격과 드론 프로토콜 행동
- 채굴
- 모래 시뮬레이션
- HP 재생
- XP/골드 획득
- 실패 조건 체크

### 11-2. Day 종료

Day 1~29에서 시간이 끝나면 결과 화면으로 가지 않고 intermission으로 진입한다.

흐름:

1. Day active 종료
2. 일반 스폰 중단
3. intermission 진입
4. 마지막 활성 블록 정리 대기
5. `1.25초` 뒤 키오스크 투하
6. 키오스크 투하 직후 중앙 상단 유해물질 카운트다운 시작
7. `10초` 뒤 `유해물질 경고`, 다시 `10초` 뒤 `유해물질 위험`, 다시 `10초` 뒤 글리치 위험 상태 진입
8. 키오스크 상호작용으로 상점 진입

intermission 유해물질 단계:

| 단계 | 지속시간 | 피해 | 회복 |
|---|---:|---:|---|
| 카운트다운 | `10초` | 없음 | 허용 |
| `유해물질 경고` | `10초` | 초당 `2` 고정 피해 | 정지 |
| `유해물질 위험` | `10초` | 초당 `5` 고정 피해 | 정지 |
| 글리치 위험 | Next Day까지 | 초당 `10` 고정 피해 | 정지 |

상점 UI 또는 ESC pause처럼 게임 진행이 정지된 동안에는 유해물질 타이머와 피해도 멈춘다.

### 11-3. 키오스크

키오스크는 월드 오브젝트로 낙하한다.

| 항목 | 값 |
|---|---:|
| 상호작용 키 | `E` |
| 상호작용 거리 | `2U` |
| 낙하 속도 | `780 px/s` |
| 투하 지연 | `1.25초` |

### 11-4. 상점

상점은 intermission 중 진입 가능한 실제 구매 UI다.

현재 기능:

- 상점 아이템 5개 롤
- 아이템 목록 표시
- 상세 정보 표시
- 가격 표시
- 구매 가능 여부 표시
- 골드 차감
- 구매 성공 시 현재 상점 목록에서 제거
- `Close`
- `Next Day`

신규 장비 카테고리:

- `weapon`
- `drone`
- `drone_protocol`
- `passive_module`

원본 데이터의 `attack_module`, `function_module`, `enhance_module`은 상점 런타임에서 각각 `weapon`, `drone_protocol`, `passive_module`로 정규화한다.

장착 규칙:

- 무기는 빈 좌·우 슬롯을 먼저 채우고, 가득 찬 경우 선택 슬롯을 교체한다.
- 드론 프로토콜은 5슬롯까지 중복 장착할 수 있고, 가득 찬 경우 선택 슬롯을 교체한다.
- 패시브 모듈은 5슬롯까지 장착한다. 교체와 효과 재계산은 후속 작업이다.

### 11-5. Next Day

`Next Day` 버튼을 누르면:

1. 페이드 아웃
2. 이자 지급
3. 예약된 벽 복구가 있으면 실행
4. intermission 유해물질 상태와 HUD 초기화
5. intermission 종료
6. Day 증가
7. 새 Day 시간 설정
8. 스폰 타이머 재시작
9. 페이드 인

기본 규칙:

- 벽은 자동 복구하지 않는다.
- 모래도 기본적으로 유지된다.
- 벽 복구는 특정 효과가 예약했을 때만 실행된다.

## 12. 상점 아이템 시스템

### 12-1. 카테고리

| 카테고리 | 의미 |
|---|---|
| `weapon` | 좌측/우측 슬롯에 장착하는 무기 |
| `drone` | 등급 없는 기본 드론. 현재는 교체 상품으로 판매하지 않음 |
| `drone_protocol` | 기본 드론이 자동 실행하는 행동 |
| `passive_module` | 무기와 프로토콜에 추가 효과를 부여하는 장비 |

### 12-2. 랭크

아이템 랭크:

- D
- C
- B
- A
- S

Day가 높아질수록 높은 랭크 비중이 증가한다.
행운은 랭크 weight를 보정한다.

### 12-3. 구매 결과

- 무기: 좌측/우측 슬롯 장착 또는 교체
- 드론 프로토콜: 5슬롯 장착. 동일 프로토콜 중복 가능
- 패시브 모듈: 5슬롯 장착. 일부 효과는 중복 적용 불가

## 19. 설계 원칙 요약

- 코어 루프를 깨지 않는다.
- 상점은 현재 구현된 시스템으로 취급한다.
- 공격은 무기와 드론 프로토콜 중심으로 설명한다.
- legacy 공격모듈 5개 장착 구조와 신규 무기 2개 구조를 동시에 활성화하지 않는다.
- 블록은 material x size + optional type으로 설명한다.
- 모래는 단순 이펙트가 아니라 실패 조건과 전장 압박의 핵심이다.
- Day 전환에서 벽과 모래는 기본 유지한다.
- 메타 시스템은 placeholder 범위를 명확히 표시한다.
