# Burial Protocol - Run HUD / UI 개선 문서

## 0. 목적

이 문서는 현재 `HUD.gd`, `PauseMenu.gd`, `DayShopUI.gd` 기준으로
런 중 UI가 어떤 정보를 어떻게 보여주는지 정리한다.

기준일: `2026-04-20`

현재 런 UI는 단순 표시를 넘어서,
Day 진행/생존/경제/스킬/스탯 확인까지 담당한다.

---

## 1. 현재 런 UI 구성

현재 런 중 플레이어가 보는 주요 UI는 아래와 같다.

- 좌측 상단: Day 진행 HUD
- 좌측 중단: 세로 센서 HUD
- 좌측 하단: 플레이어 상태 HUD
- 우측 상단: 경제/중량 HUD
- 우측 하단: 스킬 슬롯 3칸
- 전체 화면 오버레이: 레벨업 UI, 상점 UI, 일시정지 UI, 페이드

---

## 2. 좌측 상단 - Day 진행 HUD

표시 정보:

- 현재 Day / 총 Day
- 현재 난이도
- 다음 보스 Day 예고
- 남은 시간 텍스트
- 남은 시간 바

특징:

- 시간이 `10초 이하`면 시간 바 색상이 붉게 변함
- Day 타입 자체를 직접 노출하지는 않지만, 다음 보스 경고를 계속 표시

역할:

- 현재 전투 템포 파악
- 보스 Day 도달 시점 예측
- 남은 시간 압박 전달

---

## 3. 좌측 중단 - 세로 센서 HUD

현재 센서는 미니맵이 아니라 "중앙 전장 수직 스캐너"에 가깝다.

표시 요소:

- 카메라 뷰포트 구간
- 활성 낙하 블록 위치
- 플레이어 위치

범위:

- 카메라 기준 위쪽 약 2스크린
- 아래쪽 약 1스크린
- 가로는 중앙 전장 10칸만 사용

역할:

- 위에서 내려오는 블록 위치를 빠르게 파악
- 현재 전장 내 상대적 높이 감각 제공

---

## 4. 좌측 하단 - 플레이어 상태 HUD

표시 정보:

- 현재 레벨
- HP 바와 수치
- 배터리 바와 수치
- XP 바와 수치
- 디버그 패널(`Tab`)

### 4-1. HP

- 현재 체력 / 최대 체력
- 저체력 구간에서 바 색상 경고

### 4-2. 배터리

- 벽타기용 자원
- Player getter를 통해 실시간 반영
- 벽타기 중 감소
- 벽타기 중이 아닐 때 회복

### 4-3. XP

- 현재 XP / 다음 레벨 요구치
- 레벨업 직후 즉시 갱신

---

## 5. 우측 상단 - 경제 / 중량 HUD

표시 정보:

- 총 골드
- 이번 Day 골드 변화량
- 현재 중량 / 최대 중량
- 중량 바
- 중량 상태 텍스트

### 5-1. 중량 상태 단계

- `SAFE`
- `SLOW`
- `CRUSH`
- 임계 직전 `CRUSH!` 점멸

### 5-2. 표시 단위

내부 실패 기준은 모래 셀 수이지만,
HUD는 아래 표시 스케일을 사용한다.

- 내부 한도: `2400` 셀
- 표시 환산: 셀당 `0.1 KG`
- HUD 최대 표시: `240.0 KG`

즉, 판정 단위와 표시 단위를 분리해 숫자가 과도하게 커 보이지 않도록 했다.

---

## 6. 우측 하단 - 스킬 슬롯 UI

현재 슬롯은 3칸 고정 구조다.

### 6-1. 1번 슬롯

- 스킬명: `DASH`
- 아이콘 텍스트: `D`
- 입력 힌트: `Z`
- 실제 대시 쿨다운과 연동

표시 요소:

- 사용 가능 여부
- 어두운 오버레이
- 남은 쿨다운 숫자

### 6-2. 2, 3번 슬롯

- `EMPTY`
- placeholder
- 향후 스킬 확장용 자리

중요:
현재 구조는 과도한 일반화 없이,
기존 HUD 안에 3칸 슬롯을 안전하게 붙인 상태다.

---

## 7. ESC 일시정지 / 스탯 확인 UI

현재 ESC는 토글형 일시정지 메뉴다.

- 닫힘 상태에서 ESC -> 열림
- 열림 상태에서 ESC -> 닫힘

구성:

- 좌측: 계속하기 / 메인 허브로 / 게임 종료 / 설정(placeholder)
- 우측: 현재 스탯 리스트

### 7-1. 우측 스탯 리스트

현재 표시 항목:

- 공격력
- 공격속도(`/sec`)
- 공격범위
- 치명타 확률
- 치명타 배율
- 현재 체력 / 최대 체력
- 방어력
- HP 재생
- 이동속도
- 점프력
- 채굴 데미지
- 채굴 속도(`/sec`)
- 채굴 범위
- 이자율
- 행운

행운은 현재 "효과 없음"으로 표기된다.

---

## 8. 상점 UI

`DayShopUI`는 현재 최소 연결형 UI다.

표시 목적:

- intermission 중 상점 단계 진입을 명확히 보여줌
- `Next Day` 전환 게이트 제공

현재 버튼:

- `Close`
- `Next Day`

아직 없는 것:

- 구매 목록
- 실제 상점 아이템
- 가격/효과 상세 패널

---

## 9. 현재 HUD/UI가 잘 하고 있는 점

- Day 압박, 경제, 생존 상태가 동시에 보인다
- 배터리와 대시 쿨다운처럼 새 시스템의 상태가 즉시 반영된다
- ESC 메뉴에서 최종 스탯을 한 번에 확인할 수 있다
- 중량을 표시 스케일로 재가공해 읽기 쉬워졌다
- 스킬 UI 확장 자리가 이미 준비되어 있다

## 10. 현재 제한점과 TODO

### 10-1. HUD 본문 문자열

일부 HUD 라벨은 여전히 영어 기반이다.

예:

- `CURRENT DAY`
- `TIME REMAINING`
- `GOLD`
- `WEIGHT LOAD`
- `SAFE`, `SLOW`, `CRUSH`

이 부분은 추후 `Locale.gd`까지 정리하는 단계에서 맞춰도 된다.

### 10-2. 상점 UI

현재는 게이트 역할만 한다.
실제 구매 UI 확장은 다음 단계 과제다.

### 10-3. 세로 센서 HUD

실용성은 높지만, 위험도 구분이나 보스 강조 같은 추가 연출은 아직 약하다.

---

## 11. 정리

현재 런 UI는 아래 역할을 모두 수행하고 있다.

- Day 진행 정보 전달
- 생존 자원 표시
- 경제/중량 경고
- 대시 쿨다운 표시
- intermission 전환 게이트
- ESC 기반 상세 스탯 확인

즉, 현재 HUD/UI는 이미 "실제 플레이 루프를 지탱하는 인터페이스" 단계에 들어와 있다.
