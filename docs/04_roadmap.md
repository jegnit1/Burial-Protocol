## 2. 최우선 작업

### 2-1. 문서 안정화

- `project_rules.md`를 새 문서 구조에 맞게 정리
- 구형 분리 스펙 제거 또는 archive 이동
- Codex가 우선 읽을 문서 순서 고정
- 문서 간 중복 내용 최소화

### 2-2. 상점 아이템 정리

- `ShopItemCatalog.tres`의 현재 아이템 목록 점검
- attack/function/enhance 카테고리별 역할 정리
- 구매 조건, 가격, 랭크 분포 밸런스 조정
- 상점 UI 텍스트 한글화
- 구매 실패 사유를 더 명확히 표시

### 2-3. 공격모듈 정리

- 기본 시작 모듈을 캐릭터 데이터로 이전
- melee/ranged/mechanic 모듈별 실제 전투 감각 점검
- 모듈 시각 공전 연출 개선
- 투사체/레이저/메카닉 피드백 강화
- 오라형 세부 동작 확정

### 2-4. 블록 데이터 정리

- material/size/type 데이터 구조 점검
- 크기별 출현 구간 밸런스 조정
- 재질별 HP/보상/모래량 밸런스 조정
- 특수 결과 구현 여부 점검
- 보스 블록 데이터 정리

## 3. 근시일 내 작업

### 3-1. HUD / UI

- HUD 문자열 한글화
- 상점 UI 가독성 개선
- 공격모듈 장착 상태 표시 개선
- 스킬 슬롯 2, 3번 활용 방향 결정
- 세로 센서 HUD 위험도 표시 추가 검토
- 중량 경고 연출 강화

### 3-2. 플레이어 감각

- 1U, 플레이어 크기, 카메라 배율 최종 점검
- 벽타기 상단 제한 플레이 감각 확인
- 대시와 모래 충돌 감각 점검
- 채굴 범위와 리듬 튜닝
- 공격모듈별 공격속도 체감 점검

### 3-3. 모래 / 중량

- 모래 총량 증가 속도 밸런스 확인
- 중량 실패까지 걸리는 평균 시간 측정
- Day별 모래 압박 곡선 조정
- 벽 복구 아이템 도입 여부 결정
